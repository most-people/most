# MostBox 赏金审计与 Fraud Proof

> 以 `项目计划书.md` 为准：MVP 不做零知识 PoSt；使用 Merkle chunk 审计、赏金猎人举报和链上 fraud proof。

---

## 一、方案选择

| 方案 | 决策 |
| ---- | ---- |
| PoSt 定期证明 | 暂不做，链上/链下复杂度高 |
| 零知识证明 | 暂不做，不符合完整副本极简路线 |
| Merkle audit | 当前主线，简单且可与完整副本模型直接结合 |

---

## 二、审计流程

```
赏金猎人                                  被审计节点
  │                                          │
  │── 1. join(topic) ───────────────────────→│
  │── 2. request chunk[i] + proof ──────────→│
  │                                          │
  │←─ 3. {chunkData, proof[], signature} ────│
  │                                          │
  │ 4. 本地验证：                            │
  │    hash(chunkData) = leaf[i]             │
  │    Merkle proof → chunkMerkleRoot        │
  │    signature 属于订单节点                │
  │                                          │
  │ 5a. 通过：无链上操作                      │
  │ 5b. 失败：提交 fraud proof 上链           │
```

正常响应只在本地验证，不产生 Gas。仅欺诈上链。

Fraud proof 采用最小上链数据：`{orderId, accusedNode, challengeNonce, chunkIndex, chunkHash, proof[], nodeSignature}`。`chunkData` 仅链下传输，不进入 calldata。

节点签名必须覆盖 `chainId, orderId, accusedNode, challengeNonce, chunkIndex, chunkHash, proofHash, deadline`。

---

## 三、Counter-proof 机制

Counter-proof 只用于反驳伪造举报，例如 `nodeSignature` 无效、challenge 过期、`orderId/chunkIndex/challengeNonce` 不匹配。若 fraud proof 已包含有效节点签名，且签名覆盖的 `chunkHash + proof[]` 无法还原到链上 `chunkMerkleRoot`，节点不能再用之后提交的正确 chunk 洗白。

---

## 四、惩罚机制

| 场景 | 处理 |
| ---- | ---- |
| fraud proof 成立 | `max(最低罚没额, 节点总质押 × 订单文件大小 / 节点声明容量 × 重复违规系数)` |
| fraud proof 为假 | 罚没赏金猎人质押 |
| counter-proof 成立 | 赏金猎人质押部分归被诬节点 |
| 单次节点不响应 | 扣当期可结算收益/信誉分 |
| 多次独立审计不响应 | 冻结收益并触发 repair；repair 证实丢失后罚没 |

罚没分配：30% 赏金猎人，70% 销毁。

---

## 五、Repair 触发

```
fraud proof 成立
  │
  ├── 标记节点 failed，按最低罚没额/比例/重复违规系数罚没
  ├── 从候选节点池选择替补
  ├── 替补 join(topic)，从存活节点全量拷贝完整文件
  ├── 替补提交 chunkMerkleRoot 确认后加入订单
  └── repair 带宽费从 Repair 储备金结算
```

## 六、性能目标

| 指标 | 目标值 | 测量方式 |
| ---- | ------ | -------- |
| 单次 chunk proof 生成 | ≤ 50ms | 基准测试 |
| 本地 proof 验证 | ≤ 20ms | 单元测试 |
| 正常审计 Gas | 0 | 无链上操作 |
| fraud proof Gas | 测试网评估后确定 | Hardhat gas reporter |
