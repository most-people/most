# MostBox 赏金审计与 Fraud Proof

> 以 `项目计划书.md` 为准：MVP 不做零知识 PoSt；使用 Merkle chunk 审计、赏金猎人举报和链上 fraud proof。

---

## 一、方案选择

当前阶段采用 **Merkle Challenge-Response + Fraud Proof**：

1. 订单链上保存 `merkleRoot` 和 `chunkSize`。
2. 赏金猎人离链请求节点返回指定 `chunk[i] + Merkle proof`。
3. 正常响应只在本地验证，不产生 Gas。
4. 验证失败时提交 fraud proof 上链。
5. 合约验证后罚没作恶节点，并触发 repair。

不采用周期性零知识证明或 PoSt 的原因：

| 方案 | MVP 决策 |
| ---- | -------- |
| PoSt 定期证明 | 暂不做，链上/链下复杂度高 |
| 零知识证明 | 暂不做，不符合完整副本极简路线 |
| Merkle audit | 当前主线，简单且可与完整副本模型直接结合 |

---

## 二、审计流程

```
赏金猎人                                  被审计节点
  │                                          │
  │── 1. join(topic) ───────────────────────→│
  │── 2. request chunk[i] + proof ──────────→│
  │                                          │
  │←─ 3. {chunkData, proof[], signature} ────│
  │                                          │
  │ 4. 本地验证：                            │
  │    hash(chunkData) = leaf[i]             │
  │    Merkle proof → merkleRoot             │
  │    signature 属于订单节点                │
  │                                          │
  │ 5a. 通过：无链上操作                      │
  │ 5b. 失败：提交 fraud proof 上链           │
```

### 3.2 `FraudProofModule.sol`

```solidity
struct FraudProof {
    bytes32 orderId;
    address accusedNode;
    uint256 chunkIndex;
    bytes chunkData;
    bytes32[] merkleProof;
    bytes nodeSignature;
}

contract FraudProofModule {
    function submitFraudProof(FraudProof calldata proof) external;
    function submitCounterProof(FraudProof calldata proof) external;
}
```

链上验证内容：

- `accusedNode` 是订单存储节点。
- `chunkData` 与 `merkleProof` 无法还原到订单 `merkleRoot` 时，fraud claim 成立。
- `nodeSignature` 用于证明响应来自被审计节点。
- 被诬节点可提交同 chunk 的有效 counter-proof 反证。

## 四、惩罚机制

| 场景 | 处理 |
| ---- | ---- |
| fraud proof 成立 | 罚没被审节点 100% 质押 |
| fraud proof 为假 | 罚没赏金猎人质押 |
| 被诬节点 counter-proof 成立 | 赏金猎人质押部分归被诬节点 |
| 节点不响应 | 记录信誉报告，不直接罚没 |

罚没分配：

| 来源 | 赏金猎人 | 销毁 |
| ---- | -------- | ---- |
| 作恶节点质押 | 30% | 70% |

## 五、Repair 触发

```
fraud proof 成立
  │
  ├── FraudProofModule 标记节点 failed
  ├── NodeRegistry 扣除质押
  ├── RepairModule 选择替补节点
  ├── 替补 join(topic)
  ├── 从存活节点全量拷贝完整文件
  └── 替补提交 merkleRoot 确认后加入订单
```

## 六、性能目标

| 指标 | 目标值 | 测量方式 |
| ---- | ------ | -------- |
| Merkle tree 构建 | 1GB 文件可在可接受时间内完成 | 基准测试 |
| 单次 chunk proof 生成 | ≤ 50ms | 基准测试 |
| 本地 proof 验证 | ≤ 20ms | 单元测试 |
| 正常审计 Gas | 0 | 无链上操作 |
| fraud proof Gas | 测试网评估后确定 | Hardhat gas reporter |
