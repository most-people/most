# MostBox 零知识存储证明

> Proof-of-Spacetime (PoSt) 方案、Merkle tree、Challenge-Response 机制

---

## 一、证明方案选择

采用 **Proof-of-Spacetime (PoSt)** 变体：

1. **Challenge-Response 模式**：
   - 合约随机生成挑战（随机偏移量 + 长度）
   - 节点读取对应位置数据
   - 计算 Merkle root 并提交
   - 合约验证 root 与承诺一致

2. **不泄露内容**：
   - 证明只涉及 Merkle tree 路径
   - 不传输实际数据内容

---

## 二、证明流程

```
合约                        节点
  │                          │
  ├── ① 随机挑战 ───────────→│
  │   (random offset, size)  │
  │                          ├── ② 读取数据块
  │                          ├── ③ 计算 Merkle root
  │←── ④ 提交证明 ───────────┤
  │   (merkleRoot, signature)│
  ├── ⑤ 验证 ────────────────│
  │   ✓ 通过 / ✗ 失败        │
  │                          │
  ├── ⑥ 结算收益 / 惩罚 ────→│
```

## 四、惩罚机制（Slashing）

| 违规行为          | 惩罚                      |
| ----------------- | ------------------------- |
| 证明超时          | 扣当日收益                |
| 证明验证失败      | 扣 10% 质押金             |
| 多次证明失败      | 扣 50% 质押金 + 降级      |
| 数据丢失/拒绝提供 | 扣除全部质押金 + 永久封禁 |
