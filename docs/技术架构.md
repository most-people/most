## 一、分层架构

```
┌─────────────────────────────────────────────────────────────┐
│                      用户交互层                              │
│  Next.js 16 (Web)  │  Electron (Desktop)  │  CLI (未来)     │
├─────────────────────────────────────────────────────────────┤
│                      应用服务层                              │
│  文件管理  │  聊天频道  │  节点运营  │  订单管理  │  钱包    │
├─────────────────────────────────────────────────────────────┤
│                      内容完整性层                            │
│  UnixFS CID v1  │  256KB chunk  │  chunkMerkleRoot  │  本地校验  │
├─────────────────────────────────────────────────────────────┤
│                      P2P 网络层                              │
│  Hyperswarm 4.x  │  Hyperdrive 13.x  │  Hypercore 17.x     │
├─────────────────────────────────────────────────────────────┤
│                      区块链层                                │
│  USDC/USDT 支付  │  MOST 质押  │  订单合约  │  Paymaster    │
└─────────────────────────────────────────────────────────────┘
```

## 二、技术栈

| 层级   | 技术                             | 作用                                   |
| ------ | -------------------------------- | -------------------------------------- |
| 网络层 | Hyperswarm 4.x                   | 节点发现、NAT 打洞、P2P 组网           |
| 存储层 | Hyperdrive 13.x + Hypercore 17.x | 完整文件副本存储、内容寻址             |
| 校验层 | CID v1 + Merkle Tree             | 文件身份、chunk 完整性、链上验证锚     |
| 合约层 | Base (Solidity)                  | 订单、质押、赏金审计、结算、修复       |
| 交互层 | Next.js 16 + React 19            | Web UI + 桌面客户端                    |
| 部署层 | Docker + Docker Compose          | 存储节点一键部署                       |

### CID 生成参数

MostBox v1 复用当前 `server/src/core/cid.js` 实现：`ipfs-unixfs-importer@16.1.5`、`cidVersion: 1`、`rawLeaves: true`、`wrapWithDirectory: false`。未显式写出的 importer 默认参数由 `package-lock.json` 锁定，升级依赖前必须通过 CID 黄金样本测试。

## 五、非 MVP 能力

| 能力 | 当前决策 |
| ---- | -------- |
| 默认应用层加密 | 暂不内置；用户可自行本地加密后上传 |
| 文件分片/纠删码 | 暂不实现；MVP 采用完整副本 `m>=2` |
| x402 | 暂不考虑；MVP 仍以链上订单和稳定币托管为主 |
| 零知识证明 | 暂不实现；MVP 使用 Merkle 审计 + fraud proof |
