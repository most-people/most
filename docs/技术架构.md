## 二、技术栈

| 层级   | 技术                             | 作用                                   |
| ------ | -------------------------------- | -------------------------------------- |
| 网络层 | Hyperswarm 4.x                   | 节点发现、NAT 打洞、P2P 组网           |
| 存储层 | Hyperdrive 13.x + Hypercore 17.x | 完整文件副本存储、内容寻址             |
| 校验层 | CID v1 + Merkle Tree             | 文件身份、chunk 完整性、链上验证锚     |
| 合约层 | Base (Solidity)                  | 订单、质押、赏金审计、结算、修复       |
| 交互层 | Next.js 16 + React 19            | Web UI + 桌面客户端                    |
| 部署层 | Docker + Docker Compose          | 存储节点一键部署                       |

### 核心技术特性

1. **纯 P2P 组网** — 无 Tracker、无网关、无中心协调服务，节点直连。
2. **完整副本模型** — MVP 使用 `m=2`，每个质押节点持有完整文件副本。
3. **内容寻址** — 文件使用现有 IPFS UnixFS CID v1；Hyperswarm topic 取 `rootCid.multihash.digest`。
4. **Merkle 完整性校验** — 文件按 256KB chunk 构建 Merkle root，链上仅保存验证锚。
5. **CID 即权限** — 知道 CID 即可查询订单、定位节点、下载文件、为订单续费。
6. **可选本地加密** — MostBox MVP 不默认加密；隐私文件由上传者在进入 MostBox 流程前自行加密。

## 三、数据流架构

### 3.1 文件上传与下单

```
原始文件
  │
  ├── 计算 UnixFS CID v1
  ├── 按 256KB chunk 构建 Merkle root
  ├── 写入上传者 Hyperdrive，key = /<cid>
  ├── join(rootCid.multihash.digest)
  └── 链上下单：{CID, merkleRoot, chunkSize, m=2, stablecoin, duration}
        │
        └── emit OrderCreated
```

### 3.2 存储节点抢单

```
存储节点监听 OrderCreated
  │
  ├── join(topic)
  ├── 从上传者 pull 完整文件
  ├── 本地重算 Merkle root
  ├── submit pullConfirm
  └── 前 m 个确认者成为订单存储节点
```

### 3.3 下载与做种

```
下载者仅知道 CID
  │
  ├── 查询链上订单与节点列表
  ├── join(topic)
  ├── 从任意持有完整副本的 peer 下载
  ├── 校验 Merkle root 与 CID
  └── 下载完成后继续在 topic 中临时做种
```

## 四、Base 链 + ERC-4337 集成

### 4.1 账户系统：mostWallet 确定性钱包

**用户账户基于现有 `mostWallet.js` 派生，不引入外部钱包或社交登录作为 MVP 前提。**

```
username + password
    ↓ PBKDF2 (50,000 次迭代, sha512)
32 bytes seed
    ↓ SHA256 + BIP-39
助记词 (mnemonic)
    ↓ BIP-44 (m/44'/60'/0'/0/0)
EVM 地址 (Base 链)
    ↓
合约交互 / 签名交易
```

### 4.2 链选型理由

- Gas 低，适合订单、确认、结算等小额高频操作。
- EVM 生态成熟，可复用 Solidity、ethers.js、OpenZeppelin 工具链。
- ERC-4337 Paymaster 可在后续阶段为用户代付 Gas，降低链上交互门槛。

| 能力 | 当前决策 |
| ---- | -------- |
| 默认应用层加密 | 暂不内置；用户可自行本地加密后上传 |
| 文件分片/纠删码 | 暂不实现；MVP 采用完整副本 `m=2` |
| 动态定价 | 暂不实现；MVP 使用 `$0.1/GB/月` 固定定价 |
| 零知识证明 | 暂不实现；MVP 使用 Merkle 审计 + fraud proof |
