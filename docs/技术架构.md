## 二、技术栈

| 层级   | 技术                             | 作用                         |
| ------ | -------------------------------- | ---------------------------- |
| 网络层 | Hyperswarm 4.x                   | 节点发现、NAT 打洞、P2P 组网 |
| 存储层 | Hyperdrive 13.x + Hypercore 17.x | 加密分片存储、内容寻址       |
| 加密层 | AES-256-GCM + 零知识证明         | 端到端加密、存储证明         |
| 合约层 | Base (Solidity)                  | 订单管理、质押、结算、惩罚   |
| 交互层 | Next.js 16 + React 19            | Web UI + 桌面客户端          |
| 部署层 | Docker + Docker Compose          | 节点一键部署                 |

### 核心技术特性

1. **纯 P2P 组网** — 无 Tracker、无网关、无共识节点，节点直连
2. **客户端全加密** — 本地 AES-256 加密，密钥仅用户持有
3. **元数据全加密** — 文件名、路径、大小、类型全部加密
4. **零知识存储证明** — 节点证明"存了数据"但不泄露内容
5. **多副本 + 纠删码** — 分片存储多节点，单节点下线不影响可用性

## 三、数据流架构

### 3.1 文件上传流程

```
用户设备（加密）                    P2P 网络                    存储节点
     │                               │                           │
     ├── ① AES-256 加密文件 ────────→│                           │
     ├── ② 生成 CID + 分片 ─────────→│                           │
     ├── ③ 创建订单（合约）──────────→├── ④ 匹配节点 ────────────→│
     ├── ⑤ 发送加密分片 ─────────────→├── ⑥ 接收存储 ───────────→│
     │                               │←── ⑦ 提交存储证明 ────────┤
     │←── ⑧ 订单生效通知 ────────────┤                           │
     │                               │                           │
     │←── ⑨ 定期收益结算 ────────────┤←── ⑩ 自动结算 ────────────┤
```

### 3.2 应用数据流

```
客户端请求 → Hono API 路由 → MostBoxEngine → Corestore/Hyperdrive → P2P 复制
                ↓
          WebSocket 事件推送 → 客户端实时更新
                ↓
          Base 链合约交互（通过 Paymaster 代付 Gas）
```

- 前端通过 REST API 与后端交互
- 实时通知通过 WebSocket (`/ws`) 推送
- P2P 数据复制由 Hyperswarm 自动处理
- 链上操作通过 ERC-4337 Paymaster 代付，用户无感

## 四、Base 链 + ERC-4337 集成

### 4.1 账户系统：mostWallet 确定性钱包

**用户账户完全基于现有 `mostWallet.js` 派生，不引入外部钱包或社交登录。**

```
username + password
    ↓ PBKDF2 (50,000 次迭代, sha512)
32 bytes seed (danger)
    ↓ SHA256 + BIP-39
助记词 (mnemonic)
    ↓ BIP-44 (m/44'/60'/0'/0/0)
EVM 地址 (Base 链)
    ↓
合约交互 / 签名交易
```

**核心优势**：

- 用户只需记住 username + password，无需助记词/私钥
- 确定性派生：同一凭据永远生成同一钱包
- 密钥本地派生，不上传服务器
- 兼容 EVM 生态，可直接与 Base 链合约交互

### 4.2 链选型理由

**选择 Base 的理由**：

- Gas 极低（~$0.01/笔），项目方可承担代付成本
- ERC-4337 Paymaster 生态成熟，代付 Gas 实现简单
- Coinbase 生态协同，未来 UX 优化空间大
- EVM 兼容，ethers.js 无缝对接

**代付 Gas 架构**：

```
用户操作 (username + password)
    ↓
mostWallet 派生签名者
    ↓
交易提交至 Bundler
    ↓
Paymaster 验证并代付 Gas (项目方预存 ETH)
    ↓
Base 链执行合约
```

用户完全无感，无需持有 ETH，无需理解 Gas 概念。
