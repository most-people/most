# MostBox 实施路线图

> 分阶段实施计划、任务清单、里程碑与验证标准

---

## 阶段 1：应用层加密（2 周）

| 任务                 | 产出                            | 验证                         |
| -------------------- | ------------------------------- | ---------------------------- |
| 实现 `encryption.js` | AES-256-GCM 加密/解密           | 单元测试：加密→解密=原始数据 |
| 实现元数据加密       | `published-files.json` 加密存储 | 集成测试：上传→存储→下载     |
| 改造 `publishFile`   | 上传时自动加密                  | 端到端测试                   |
| 改造 `downloadFile`  | 下载时自动解密                  | 端到端测试                   |
| 前端加密状态提示     | UI 反馈                         | 手动测试                     |

**里程碑**：v0.2.0 — 所有新文件自动加密，现有文件保持兼容

---

## 阶段 2：智能合约基础（3 周）

| 任务                     | 产出                     | 验证                       |
| ------------------------ | ------------------------ | -------------------------- |
| 初始化 Hardhat 项目      | `contracts/` 目录 + 配置 | `npx hardhat compile` 通过 |
| 实现 `MostToken.sol`     | ERC-20 代币合约          | 单元测试：铸造/转账/授权   |
| 实现 `NodeRegistry.sol`  | 节点注册+质押            | 单元测试：注册/质押/退出   |
| 实现 `StorageMarket.sol` | 订单创建+匹配            | 单元测试：创建/匹配/完成   |
| 部署到 Base Sepolia      | 合约地址 + 验证          | Basescan 验证通过          |
| 编写部署脚本             | `scripts/deploy.js`      | 一键部署                   |

**里程碑**：v0.3.0-alpha — 测试网合约部署完成，可创建/匹配订单

## 总体进度

```
v0.1.0 (当前)     →  v0.2.0 (加密)     →  v0.3.0 (合约)     →  v0.4.0 (支付)
P2P 文件分享          应用层加密           智能合约集成         订单支付
P2P 聊天              元数据加密           节点运营            事件监听
Web3 工具箱           密钥管理扩展         订单匹配            链上确认

v0.5.0 (冗余)     →  v0.6.0 (证明)     →  v1.0.0 (完整)
多副本存储            零知识证明           付费存储网络
纠删码                链上验证             动态定价
数据修复              Slashing             社区治理
```
