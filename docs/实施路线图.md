# MostBox 实施路线图

> 以 `项目计划书.md` 为准：先完成完整副本付费存储网络，再评估加密、纠删码、动态定价等后续能力。

---

## Phase 1 — 基础数据管道（预计 2 周）

目标：跑通 `Merkle → Hyperswarm 上传/下载 → CID 校验` 全流程，先在本地/模拟环境验证。

| 任务 | 产出 | 验证 |
| ---- | ---- | ---- |
| Merkle 树构建与验证 | 256KB chunk 的 `merkleRoot` 与 proof 工具 | 单元测试：任意 chunk proof 可验证到 root |
| CID 计算对齐 | 复用现有 UnixFS CID v1 计算 | 单元测试：同一文件 CID 稳定一致 |
| Hyperdrive 完整副本写入 | key 使用 `/<cid>` 存储完整文件 | 集成测试：上传后可按 CID 读取 |
| Hyperswarm topic 下载 | topic 使用 `rootCid.multihash.digest` | 双节点本地测试：下载节点可 pull 文件 |
| 下载完整性校验 | 下载后重算 Merkle root 与 CID | 集成测试：篡改文件必须失败 |

**里程碑**：v0.2.0 — 无链环境下可通过 CID 完成 P2P 上传、下载、校验与临时做种。

---

## Phase 2 — 链上订单（预计 2-3 月）

目标：实现稳定币托管订单、节点质押注册、先到先得抢单与到期结算。

| 任务 | 产出 | 验证 |
| ---- | ---- | ---- |
| Hardhat 项目初始化 | `contracts/` + 部署脚本 | `npx hardhat compile` 通过 |
| MOST Token | ERC-20 治理/质押代币 | 单元测试：铸造、转账、销毁 |
| NodeRegistry | 节点注册、MOST 质押、容量声明 | 单元测试：注册、质押、退出 |
| OrderFactory | `{CID, merkleRoot, chunkSize, m, duration}` 订单 | 单元测试：创建订单并托管 USDC |
| pullConfirm 抢单 | 前 m 个完成 pull 并验证 root 的节点获得订单 | 并发测试：第 m+1 个确认被拒绝 |
| 到期结算 | USDC 释放给 m 个节点 | 时间推进测试：到期后可结算 |
| 前端集成 | 下单、订单状态、节点管理 UI | 手动测试：从上传到订单生效闭环 |

**里程碑**：v0.3.0 — Base 测试网上可创建付费订单，节点可通过 pullConfirm 获得订单并到期领取 USDC。

---

## Phase 3 — 赏金审计（预计 2-3 月）

目标：用赏金猎人模型替代中心化审计，仅 fraud proof 上链。

| 任务 | 产出 | 验证 |
| ---- | ---- | ---- |
| Merkle Proof 挑战-响应 | 离线 audit 协议 | 单元测试：随机 chunk proof 可通过 |
| 节点响应签名 | 节点对审计响应签名 | 单元测试：伪造响应无法通过 |
| FraudProofModule | 链上验证 fraud proof 并罚没 | 合约测试：坏 proof 成立、好 proof 失败 |
| Counter-proof | 被诬节点提交同 chunk 自证 | 合约测试：诬告者质押被罚 |
| 信誉分记录 | 多人报告、失败、恢复状态记录 | 集成测试：状态转移符合规则 |

**里程碑**：v0.4.0 — 任何参与者都可审计订单节点；欺诈上链后触发罚没。

---

## Phase 4 — 修复与经济治理（预计 2-3 月）

目标：节点失败后自动补位，并完成手续费分配、回购销毁和 Paymaster。

| 任务 | 产出 | 验证 |
| ---- | ---- | ---- |
| RepairModule | 失败节点替换、替补加入订单 | 集成测试：模拟节点失败后恢复到 m 个副本 |
| 全量修复流程 | 替补从存活节点拷贝完整文件 | 集成测试：新节点校验 Merkle root 后加入 |
| Repair 储备金 | repair 带宽费结算 | 合约测试：支出来源和上限正确 |
| MOSTBurner | 手续费 70% 回购销毁 MOST | 合约测试：手续费分配为 70%/30% |
| ERC-4337 Paymaster | 用户无 ETH 也可完成指定交易 | 测试网验证：代付下单/续费交易 |

**里程碑**：v0.5.0 — 网络可自动修复副本，并完成手续费销毁与基础 Gas 代付。

---

## Phase 5 — 代币与生态（预计 2 月）

目标：从测试网验证进入主网与社区运营。

| 任务 | 产出 | 验证 |
| ---- | ---- | ---- |
| MOST 主网部署 | ERC-20 合约与分配地址 | 合约验证通过 |
| 社区空投 | 节点/开发者/早期用户激励 | 发放清单可审计 |
| 流动性池 | MOST/USDC 基础流动性 | 交易与回购路径可用 |
| 治理投票 | 参数提案与投票流程 | 测试提案完整执行 |
| 公测运营 | 节点招募与用户反馈 | 指标：有效节点数、存储量、成功下载率 |

**里程碑**：v1.0.0 — 完整付费存储网络公测启动。

## 总体进度

```
v0.1.0 (当前)       →  v0.2.0 (数据管道)  →  v0.3.0 (链上订单)
P2P 文件分享            Merkle/CID 校验        USDC 托管
P2P 聊天                topic 下载/做种        MOST 质押
Web3 工具箱             完整副本存储           pullConfirm 抢单

v0.4.0 (赏金审计)   →  v0.5.0 (修复经济)  →  v1.0.0 (公测)
Fraud proof             自动 repair            MOST 主网
Counter-proof           MOSTBurner             社区空投
Slashing                Paymaster              治理投票
```

## 暂不进入主线的能力

| 能力 | 处理方式 |
| ---- | -------- |
| 默认应用层加密 | 用户可自行本地加密；内置加密不作为 MVP 任务 |
| 分片与纠删码 | 作为未来 V2 研究，不进入当前路线图 |
| 动态定价 | MVP 固定 `$0.1/GB/月`，后续由治理调整 |
| 零知识证明 | MVP 使用 Merkle audit + fraud proof，不做 PoSt/ZK |
