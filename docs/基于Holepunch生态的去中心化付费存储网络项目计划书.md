# 基于Holepunch生态的去中心化付费存储网络项目计划书

**项目名称**：MostBox
**核心定位**：纯 P2P、零知识、无中心管控的分布式付费存储市场
**技术底座**：Hyperswarm \+ Hyperdrive \+ 智能合约
**项目愿景**：打造一个 \&\#34;用户拥有数据、节点贡献资源、代码执行规则\&\#34; 的下一代互联网存储基础设施

## 一、项目概述

### 1\.1 解决的核心痛点

- 传统网盘：中心化存储、数据泄露风险、平台删档、限速限流、隐私无保障

- 现有去中心化存储：半中心化架构、节点准入门槛高、存储方可见数据特征、使用体验差

- 市场空白：缺乏**纯 P2P 原生、存储方零知情权、人人可参与**的通用型付费存储网络

### 1\.2 核心价值主张

- **绝对隐私**：客户端先加密再上传，存储节点全程仅能看到密文，连文件名、文件大小都无法获知

- **完全去中心化**：无中心服务器、无平台管控、规则由智能合约执行，任何人无法篡改或关停网络

- **零门槛参与**：普通用户打开浏览器即可存文件，任何人可部署节点贡献硬盘带宽赚取收益

- **高效低成本**：P2P 直连传输无中转，资源按需定价，比传统网盘成本低 30%\-50%

## 二、技术架构设计

### 2\.1 整体技术栈

|层级|技术选型|核心作用|
|---|---|---|
|底层网络层|Hyperswarm 4\.x|节点自动发现、NAT 打洞、P2P 网状组网|
|分布式存储层|Hyperdrive 13\.x \+ Hypercore 17\.x|加密文件分片存储、内容寻址、版本控制|
|加密安全层|AES\-256\-GCM \+ 零知识证明 \(ZKP\)|客户端端到端加密、存储证明不泄露内容|
|合约与激励层|Arbitrum/Solana 智能合约|订单管理、节点质押、自动结算、违规惩罚|
|用户交互层|Next\.js 16 \+ React 19|网页端用户界面、桌面客户端 \(Eletron\)|
|部署运维层|Docker \+ Docker Compose|节点一键部署、容器化运行|

### 2\.2 核心技术特性

1. **纯 P2P 组网**：无 Tracker、无网关、无链上共识节点，节点直接互联

2. **客户端全加密**：文件在用户设备本地完成 AES\-256 加密，密钥仅用户持有

3. **元数据全加密**：文件名、路径、大小、类型全部加密，节点无任何可推断特征

4. **零知识存储证明**：节点证明 \&\#34;存了数据\&\#34; 但无需透露任何数据内容

5. **多副本 \+ 纠删码**：文件分片存储在多个节点，单个节点下线不影响数据可用性

## 三、网络角色与核心业务流程

### 3\.1 网络参与角色

1. **存储用户**

    - 上传加密文件，按容量 / 时长支付费用

    - 持有文件私钥，可随时取回、分享文件

    - 可设置文件分享权限，支持付费下载

2. **存储节点**

    - 部署客户端，贡献硬盘空间和网络带宽

    - 自动接收存储订单，托管加密文件分片

    - 按存储量、在线率、带宽贡献赚取收益

    - 需质押一定数量代币作为保证金

3. **网络治理者**（社区自治）

    - 投票决定网络参数调整（如手续费比例、质押要求）

    - 审核重大技术升级提案

    - 无任何特殊权限，无法干预用户数据或节点运营

### 3\.2 核心业务流程

#### （1）用户上传文件流程

1. 用户打开网页端 / 客户端，选择本地文件

2. 客户端本地完成**AES\-256 加密 \+ 分片**，生成唯一 CID

3. 用户选择存储时长、冗余等级，系统计算费用

4. 用户通过智能合约支付费用，订单上链

5. 合约自动匹配符合条件的存储节点

6. 客户端通过 Hyperswarm 将加密分片发送至匹配节点

7. 节点完成存储后，提交零知识证明至合约验证

8. 验证通过，订单生效，开始计算节点收益

#### （2）节点运营流程

1. 运营者下载节点客户端，完成质押

2. 节点自动接入 Hyperswarm 网络，上报资源信息

3. 智能合约根据节点资质分配存储订单

4. 节点接收并存储加密分片，保持在线

5. 节点定期提交存储证明至合约

6. 合约验证通过后，自动结算收益至节点账户

7. 运营者可随时申请提现收益

#### （3）用户下载文件流程

1. 用户输入文件 CID 和私钥

2. 客户端通过 Hyperswarm 全网检索存储节点

3. 从多个节点并行拉取加密分片

4. 客户端本地合并分片并解密

5. 还原原始文件供用户下载

## 四、经济模型设计

### 4\.1 代币体系

- **代币名称**：MST（暂定）

- **总供应量**：10 亿枚

- **分配方案**：

    - 节点挖矿奖励：60%（逐年递减，10 年释放完毕）

    - 团队与早期贡献者：15%（4 年线性解锁）

    - 社区与生态建设：15%

    - 储备金：10%

### 4\.2 定价与收益机制

1. **动态定价**：存储价格由市场供需自动调节，全网资源紧张时上浮，充足时下调

2. **收益分配**：

    - 存储节点：获得用户支付费用的 90%

    - 网络手续费：10%（用于生态建设和储备金）

3. **节点等级体系**：

    - 按在线时长、存储量、历史信誉划分等级

    - 高等级节点优先匹配高价值订单，获得更高收益加成

4. **惩罚机制**：

    - 节点无故离线超过 24 小时，扣除当日收益

    - 丢失数据或无法提供存储证明，扣除部分或全部质押金

    - 多次违规节点将被永久封禁
