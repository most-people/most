# MostBox 项目计划书

> 纯 P2P、内容寻址、无中心管控的分布式付费存储网络

---

| 项目         | 内容                                                                  |
| ------------ | --------------------------------------------------------------------- |
| **名称**     | MostBox                                                               |
| **代币**     | USDC/USDT（支付）+ MOST（治理+价值捕获）                              |

## 一、市场痛点

### 1.1 传统网盘

| 问题       | 表现                                   |
| ---------- | -------------------------------------- |
| 数据控制权 | 平台可随时删除、审查、限制访问         |
| 隐私风险   | 服务商可扫描文件内容，存在数据泄露隐患 |

| 项目     | 核心缺陷                                             |
| -------- | ---------------------------------------------------- |

| Sia      | 自研链生态封闭，分片信息对存储方可见                 |

### 1.3 市场空白

**缺乏纯 P2P 原生、内容寻址、人人可参与的通用型付费存储网络。**

| 价值             | 实现方式                                                             |

## 三、技术架构

### 3.1 总体流程

```
原始文件
  │
  ├→ CID = hash(原始文件)
  ├→ 按 256KB 切 chunk → 构建 Merkle 树 → merkleRoot
  ├→ topic = rootCid.multihash.digest（32B Buffer）
  └→ 写入上传者 Hyperdrive → join(topic)
       │
       ├→ 链上下单：{CID, merkleRoot, chunkSize, m=2, USDC, 1 个月}
       │     → emit OrderCreated 事件
       │
       ├→ 节点监听事件 → join(topic) → pull 完整文件 → 提交 pullConfirm
       │     → 前 m 个提交的节点获得订单
       │     → 第 m+1 个及之后被拒绝
       │
       ├→ 订单生效 → 上传者离线
       ├→ 任何人可通过 CID 免费下载（下载者自动做种）
       └→ 节点掉线 → repair → 完整全量拷贝
```

### 3.2 副本模型（m=2 RAID 1）

| 参数 | 含义 |
|------|------|
| m=2 | 2 个独立节点各自持有完整文件副本 |
| 扩容系数 | 2x（总存储 = 原文 × 2） |
| 容错 | 任一节点存活即可下载 |
| 单节点持有 | 完整文件 |
| 下载来源 | 任意 1 个节点 |

对比纠删码方案的取舍：

|  | 纠删码 (k=4,n=8) | 副本 (m=2) |
|---|---|---|
| 编码依赖 | Reed-Solomon WASM | 无 |
| Merkle 树 | 8 棵分片树 + 1 棵全局树 | 1 棵 |
| 链上存储 | globalRoot + pieceRoots[8] | merkleRoot |
| 下载逻辑 | 连多个节点 → 收 k 片 → RS 解码 | 连 1 个 → 下载 |
| 修复方式 | RS 解码重建分片 | 从存活节点全量拷贝 |
| 审计粒度 | 分片级二级验证 | 单 chunk 验证 |
| Phase 1 开发周期 | ~2 月 | ~2 周 |

质押惩罚提供了纠删码在"无信任网络"中的同等安全边际——节点质押了真金白银，删数据就是赔钱。m=2 在质押体系下耐久度充分，复杂度远低于纠删码。

### 3.3 Merkle 树

```
原始文件 [chunk_0, chunk_1, ... chunk_m]
  │
  ├─ leaf[0] = hash(chunk_0)
  ├─ leaf[1] = hash(chunk_1)
  ...
  └─ Merkle 根 ← 存储在链上 Order 合约

每 chunk = 256KB
```

一棵 Merkle 树覆盖整个文件。链上仅存 `merkleRoot` 和 `chunkSize`，不存分片信息——因为文件不分片，每个节点持有完整副本。

### 3.4 节点发现与下载

仅凭 CID 即可完成全部操作：

```
下载者（仅知 CID）
  │
  ├→ 链上查询 OrderFactory.getOrder(CID)
  │     → order.nodes[m]（ETH 地址列表）
  │     → order.merkleRoot, order.m
  │
  ├→ topic = rootCid.multihash.digest（sha2-256 的 32 字节 digest）
  │    Hyperswarm.join(topic)   // 存储节点 + 下载者临时种子聚合在此
  │
  ├→ 发起 ETH 签名挑战验证授权
  │     → 比对 order.nodes[] → 确认是合法存储节点
  │
  └→ 从任意 1 个已验证节点（或任何持有副本的 peer）下载完整文件
       → 本地构建 Merkle 树 → 比对链上 merkleRoot ✓
       → hash(文件) === CID ✓
       → 下载完成后保持连接，作为临时种子服务其他下载者
```

Topic 使用 `rootCid.multihash.digest`，sha2-256 CID 天然产 32 字节，无需额外 hash 或截断。节点重启时需重新 join 所有 held CID。

### 3.5 存储审计（赏金猎人模型）

任何网络参与者都可以审计任何订单的任意存储节点，**仅欺诈上链**。

```
赏金猎人                                  被审计节点
  │                                          │
  │── 1. join(CID) → 连接目标节点              │
  │── 2. 请求 chunk[i] + Merkle proof        │
  │                                          │
  │←─ {chunk_data, proof[]} ─────────────────│
  │                                          │
  │ 3. 本地验证：                              │
  │    hash(chunk) =? leaf[i]                 │
  │    root(leaf[i], proof[]) =? merkleRoot   │
  │                                          │
  │ 4a. 通过 → 无链上操作（零 gas）              │
  │ 4b. 验证失败 → 提交 fraud proof 上链       │
```

**Fraud proof 上链内容**：`{chunk_data, chunkIndex, proof[]}`。合约执行 `hash(chunk) → 验证 Merkle 路径 → 比对存储的 merkleRoot`。

| 场景 | 合约执行结果 | 后续 |
|------|-------------|------|
| chunk + proof 有效 | fraud claim 为假 | 罚没赏金猎人质押 → 30% 归被诬告节点 |
| chunk + proof 无效 | fraud claim 成立 | 罚没节点质押 → 30% 归赏金猎人 → 触发 repair |
| 节点不响应 | 无链上证据 | 赏金猎人无收益；连续多人报告 → 信誉降级 |

**博弈**：
- 节点作弊 → 任何赏金猎人可举报得 30% 罚没 → 作弊期望收益 < 0
- 赏金猎人诬告 → 对方提交同一 chunk 反证 → 猎人被反杀
- 无人审计 → 没有假 fraud proof 就视为诚实 → 到期正常结算

### 3.6 数据修复

```
检测：赏金猎人提交有效 fraud proof 上链
  │
  ├── 合约标记 node[i] = failed
  ├── 罚没质押（30% → 赏金猎人，70% → 销毁）
  ├── 从候选节点池随机选替补
  ├── 替补 join(CID) → 从存活节点全量拷贝完整文件
  ├── 替补本地构建 Merkle 树 → 提交 merkleRoot 上链验证
  ├── 替补加入 order.nodes[]
  └── repair 带宽费从 Repair 储备金结算
```

### 3.7 热门文件与下载者自动做种

热门文件被大量下载时，质押节点的上行带宽面临压力。解决方案遵循 B 站 WebRTC P2P 加速原理——**下载者边下边推**，和现有 Hyperswarm 行为完全一致：

```
冷文件（少数下载者）：
  下载者 A ─── pull ──→ 质押节点 X
                             ← 其他下载者直连 X 拉取

热门文件（爆发）：
  下载者 A ─── pull ──→ 质押节点 X
  下载者 B ─── pull ──→ 质押节点 X + 下载者 A（A 已完成，做种中）
  下载者 C ─── pull ──→ 质押节点 X + A + B（AB 分担流量）
  ...
  下载者 Z ─── pull ──→ X + A + B + ... + Y（100 个临时种子分摊）
```

Hyperswarm 4.x 的 peer 天然既下又传——无需额外开发，只需客户端下载完成后**不主动断连**，继续在 topic 上 serve 一段时间。

临时种子限制（客户端本地配置，不涉及链上）：
- 最多同时做种 N 个文件
- 下载完成后做种 T 小时（如 24h）
- 上传速率上限（如 1MB/s）

效果：质押节点带宽压力被临时种子分摊，热门文件反而下载更快，节点无需因"被下得多"而回避热门内容。

## 四、网络角色

### 4.1 存储用户

- 使用稳定币（USDC/USDT）支付存储费用
- 上传完整文件到 Hyperdrive → 下单 → 等待 pull 确认 → 可离线
- 通过 CID 随时取回；分享 CID 供他人免费下载
- 可自行本地加密后再存，实现访问控制

### 4.2 存储节点（矿工）

- 贡献硬盘空间和带宽，持有完整文件副本
- 监听链上 `OrderCreated` 事件，先到先得抢单，pull 文件后持续提供下载
- 赚取稳定币收益
- 需质押 MOST 作为保证金
- 声明磁盘容量决定可接订单数上限，谎报→审计发现→罚没
- 任何节点可充当"赏金猎人"审计其他节点获利

### 4.3 社区治理

- MOST 持有者投票决定副本数 m、质押量、手续费率
- 审核技术升级提案
- 无特殊权限，无法干预数据或运营

### 5.6 存储订单模型

#### 下单与抢单

```
用户充值 USDC → MostBox 钱包
      ↓
文件构建 Merkle 树 → 写入 Hyperdrive → join(topic)
      ↓
链上下单：{CID, merkleRoot, chunkSize, m=2, USDC（1 个月费用）}
      ↓
emit OrderCreated 事件 → 所有在线节点监听到
      ↓
节点 join(topic) → pull 完整文件 → 验证 merkleRoot → submit pullConfirm 上链
      ↓
合约维护计数器：前 m 个确认 → 分配为存储节点；超出的拒绝
      ↓
订单生效 → 上传者离线，节点按月领取 USDC
      ↓
到期 → 合约标记过期 → 节点释放存储
```

#### 先到先得

节点不预先分配，通过监听事件 + 抢单竞争。上传者同 topic 上 serve 源文件，任何节点 join 均可 pull。合约仅验证提交 pullConfirm 时附带的 merkleRoot 是否与订单记录一致。前 m 个提交一致 merkleRoot 的节点成为该订单的存储节点。

#### 预付与续费

每次下单预付 **1 个月**。到期后 USDC 自动释放给 m 个节点平分。续费调用 `Order.addFunds(USDC)` → `expiresAt += 1 个月`，订单继续有效。过期后可重新下单（此时节点可能已删数据，需视作新订单）。

#### 规则汇总

| 特性         | 说明 |
| ------------ | ---- |
| **预付费钱包** | 余额可转出，已下单费用不可退款 |
| **CID 即权限** | 知道 CID 即可查链上节点 → join topic → 下载 |
| **开放续费** | 任何人可为任意 CID 续费（addFunds，延长 expiresAt） |
| **节点分配** | 先到先得抢单，前 m 个 pull 确认者获得订单 |
| **Pull 确认** | 抢单节点 pull 完整文件 → 验证 merkleRoot → submit pullConfirm |
| **Pull 超时** | 下单后 24h 内未满 m 个确认 → 自动退款，节点损失由 Repair 储备金少量补贴 |
| **预付周期** | 每次 1 个月；到期前可 addFunds 续期 |
| **到期处理** | expiresAt 到达后 USDC 释放给节点，合约标记过期

## 六、智能合约架构

| 合约 | 职责 |
|------|------|
| **OrderFactory** | 创建/管理订单，托管 USDC |
| **NodeRegistry** | 节点注册、质押 MOST、信誉分 |

| **MOSTBurner** | 手续费 → DEX 买入 MOST → 销毁 |

**链上与离链分工**：

| 链上（智能合约） | 离链（节点客户端） |
|-----------------|-------------------|
| 订单创建与 USDC 托管 | Merkle 树构建 |
| 节点注册与 MOST 质押 | P2P 文件拉取与上传 |
| merkleRoot 存储（验证锚） | Hyperswarm topic 加入与发现 |
| Fraud proof Merkle 验证 | 赏金猎人 audit 挑战-响应 |
| 罚没与举报奖励分配 | 下载完整性校验（CID + Merkle） |
| 到期结算与 USDC 释放 | Repair 全量拷贝 |

## 七、竞品对比

| 项目        | 去中心化程度 | 存储方知情权 | 节点门槛      | 支付方式      |
| ----------- | ------------ | ------------ | ------------- | ------------- |
| Filecoin    | 链上去中心化 | 可见哈希     | 高（专业矿机） | FIL           |
| Storj       | 依赖 Satellite | 零知情权     | 中            | USDC/信用卡   |
| Sia         | 链上去中心化 | 可见分片信息 | 中            | Siacoin       |
| **MostBox** | **纯 P2P 原生** | **零知情权** | **极低（普通电脑）** | **USDC+MOST** |

> MostBox 不加密文件内容。如需隐私保护，上传者自行本地加密。存储方虽持有完整文件，但质押惩罚阻止其泄露或作恶。

### 差异化优势

1. **纯 P2P 原生** — 无 Satellite、无协调层，CID digest 即 Hyperswarm topic，直连即下载
2. **极简架构** — 无纠删码，无分片，完整文件副本，先到先得抢单
3. **内容寻址** — CID 即权限，无需账号体系，无需密钥分发
4. **稳定支付** — 稳定币支付，无价格波动风险
5. **开放续费** — 任何人可为任何文件续费，同一订单追加 USDC
6. **经济安全** — 质押罚没 + 赏金猎人审计，仅欺诈上链，日常零 gas
7. **下载者做种** — P2P 天然边下边推，热门文件越多人下载越快，节点带宽压力被分摊

## 八、风险与应对

| 风险 | 应对 |
|------|------|
| 节点大规模下线 | m=2 副本 + 自动 repair + 质押惩罚 |
| 赏金猎人不足（无人审计） | 罚没 30% 强激励；下载者下载时天然审计 |

| 稳定币脱锚 | 支持多稳定币（USDC/USDT/DAI） |
| 监管限制 | 开源自治，无中心服务器 |
| 早期节点不足 | 储备池补贴 + 社区空投招募 |
| 智能合约漏洞 | 分模块设计 + 第三方审计 + 时间锁升级 |
| CID 泄露 → 未授权下载 | 设计如此（CID 即权限）；自行本地加密可控访问 |
| 节点谎报容量 | 声明容量 > 实际 → 存不下 → 审计无法提供 Merkle proof → 100% 质押罚没 |
| 热门文件耗尽节点上传带宽 | 下载者自动做种（3.7），节点带宽压力被临时种子群分摊 |

## 十一、总结

MostBox 基于 Holepunch 生态的纯 P2P 传输层 + Base 智能合约，构建极简的去中心化付费存储网络。

**技术侧**：一个文件、一棵 Merkle 树、m 个副本、一个 32 字节 Hyperswarm topic（取 CID 的 multihash digest）。无纠删码，无分片，无加密。CID 即全部——标识文件、定位节点、校验完整性。节点先到先得抢单，赏金猎人审计代替代中心化验证（仅 fraud proof 上链，零日常 gas）。下载者完成即自动做种，热门文件 P2P 分摊节点带宽。作恶节点质押罚没（30% 归举报者），诚实节点签收 USDC。

**经济侧**：稳定币 + MOST 双轨模型，$0.1/GB/月定价，节点存满 1TB 月入 ~$45 USDC。手续费 70% 回购销毁 MOST，质押罚没驱动通缩。CID 可自由分享，任何人可免费下载、自行续费。网络使用驱动 MOST 销毁与节点供给增长的良性循环。
