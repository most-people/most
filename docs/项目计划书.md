# MostBox 项目计划书

> 纯 P2P、零知识、无中心管控的分布式付费存储网络

---

## 项目摘要

| 项目 | 内容 |
|---|---|
| **名称** | MostBox |
| **代币** | MOST（总量 10 亿枚） |
| **核心定位** | 用户拥有数据、节点贡献资源、代码执行规则的下一代存储基础设施 |
| **技术底座** | Hyperswarm + Hyperdrive + Arbitrum 智能合约 |
| **当前状态** | v0.1.0 已实现 P2P 文件分享、聊天频道、Web3 工具箱、桌面客户端 |

---

## 一、市场痛点

### 1.1 传统网盘

| 问题 | 表现 |
|---|---|
| 数据控制权 | 平台可随时删除、审查、限制访问 |
| 隐私风险 | 服务商可扫描文件内容，存在数据泄露隐患 |
| 体验限制 | 限速、限流、空间付费昂贵 |

### 1.2 现有去中心化存储

| 项目 | 核心缺陷 |
|---|---|
| Filecoin | 半中心化，节点需专业矿机，存储方可见明文哈希 |
| Storj | 依赖以太坊半中心化架构，节点门槛中等 |
| Sia | 自研链生态封闭，分片信息对存储方可见 |

### 1.3 市场空白

**缺乏纯 P2P 原生、存储方零知情权、人人可参与的通用型付费存储网络。**

## 二、核心价值主张

| 价值 | 实现方式 |
|---|---|
| **绝对隐私** | 客户端 AES-256-GCM 加密后上传，节点仅见密文，元数据全加密 |
| **完全去中心化** | 无中心服务器，规则由智能合约执行，无法篡改或关停 |
| **零门槛参与** | 浏览器即可存储文件，普通电脑可部署节点赚取收益 |
| **高效低成本** | P2P 直连无中转，资源按需定价，比传统网盘低 30%-50% |

## 三、技术架构

### 3.1 分层架构

```
┌─────────────────────────────────────────────────┐
│  用户交互层   Next.js 16 + React 19 + Electron  │
├─────────────────────────────────────────────────┤
│  应用服务层   文件管理 / 聊天 / 节点运营 / 钱包  │
├─────────────────────────────────────────────────┤
│  加密安全层   AES-256-GCM / 密钥派生 / ZKP      │
├─────────────────────────────────────────────────┤
│  P2P 网络层   Hyperswarm 4.x / Hyperdrive 13.x  │
├─────────────────────────────────────────────────┤
│  区块链层     Arbitrum 智能合约 / MOST 代币      │
└─────────────────────────────────────────────────┘
```

### 3.2 技术栈

| 层级 | 技术 | 作用 |
|---|---|---|
| 网络层 | Hyperswarm 4.x | 节点发现、NAT 打洞、P2P 组网 |
| 存储层 | Hyperdrive 13.x + Hypercore 17.x | 加密分片存储、内容寻址 |
| 加密层 | AES-256-GCM + 零知识证明 | 端到端加密、存储证明 |
| 合约层 | Arbitrum Solidity | 订单管理、质押、结算、惩罚 |
| 交互层 | Next.js 16 + React 19 | Web UI + 桌面客户端 |
| 部署层 | Docker + Docker Compose | 节点一键部署 |

### 3.3 核心技术特性

1. **纯 P2P 组网** — 无 Tracker、无网关、无共识节点，节点直连
2. **客户端全加密** — 本地 AES-256 加密，密钥仅用户持有
3. **元数据全加密** — 文件名、路径、大小、类型全部加密
4. **零知识存储证明** — 节点证明"存了数据"但不泄露内容
5. **多副本 + 纠删码** — 分片存储多节点，单节点下线不影响可用性

## 四、网络角色

### 4.1 存储用户

- 上传加密文件，按容量/时长支付 MOST
- 持有文件私钥，随时取回、分享
- 可设置分享权限，支持付费下载

### 4.2 存储节点

- 贡献硬盘空间和带宽
- 自动接收订单，托管加密分片
- 按存储量、在线率、带宽赚取 MOST
- 需质押 MOST 作为保证金

### 4.3 社区治理

- 投票决定网络参数（手续费、质押要求）
- 审核技术升级提案
- 无特殊权限，无法干预数据或运营

## 五、核心业务流程

### 5.1 上传文件

```
用户选择文件
  → 客户端 AES-256 加密 + 分片
  → 生成唯一 CID
  → 选择存储时长、冗余等级
  → 合约支付 MOST，订单上链
  → 合约匹配存储节点
  → P2P 发送加密分片
  → 节点提交零知识证明
  → 验证通过，订单生效
```

### 5.2 节点运营

```
部署节点客户端
  → 质押 MOST
  → 接入 Hyperswarm 网络
  → 合约分配订单
  → 接收并存储加密分片
  → 定期提交存储证明
  → 合约自动结算 MOST 收益
  → 随时提现
```

### 5.3 下载文件

```
输入 CID + 私钥
  → Hyperswarm 检索存储节点
  → 并行拉取加密分片
  → 本地合并分片
  → AES-256 解密
  → 还原原始文件
```

### 6.1 代币 MOST

### 6.2 分配方案

| 用途 | 比例 | 释放规则 |
|---|---|---|
| 节点挖矿 | 60% | 逐年递减，10 年释放完毕 |
| 团队与早期贡献者 | 15% | 4 年线性解锁 |
| 社区与生态建设 | 15% | 按需分配 |
| 储备金 | 10% | 应急与流动性 |

### 6.3 定价与收益

| 机制 | 规则 |
|---|---|
| **动态定价** | 供需调节：资源紧张上浮，充足下调 |
| **收益分配** | 节点 90% / 网络手续费 10% |
| **节点等级** | 按在线时长、存储量、信誉分级，高等级优先匹配高价值订单 |
| **惩罚机制** | 离线 >24h 扣当日收益；丢失数据扣质押金；多次违规永久封禁 |

## 八、竞品对比

| 项目 | 去中心化 | 存储方知情权 | 节点门槛 | 隐私强度 |
|---|---|---|---|---|
| Filecoin | 半去中心化 | 可见明文哈希 | 高（专业矿机） | 中 |
| Storj | 半去中心化 | 可见文件哈希 | 中 | 中 |
| Sia | 半去中心化 | 可见分片信息 | 中 | 中 |
| MaidSafe | 高度去中心化 | 零知情权 | 低 | 高 |
| **MostBox** | **近乎完全去中心化** | **绝对零知情权** | **极低（普通电脑）** | **顶级** |

### 差异化优势

1. **纯 P2P 原生** — 无中心化组件，抗审查、抗宕机
2. **绝对隐私** — 元数据全加密，存储方零知情
3. **极低门槛** — 浏览器即用，普通电脑可当节点
4. **高效轻量** — 无复杂共识，低延迟、低消耗

## 十一、总结

MostBox 基于 Holepunch 生态的纯 P2P 技术，打造真正去中心化、隐私优先的付费存储网络。用户真正拥有数据，任何人可通过贡献资源获得收益。

通过合理的经济模型和分阶段落地计划，MostBox 将成为下一代互联网存储基础设施，为 Web3 和传统用户提供安全、高效、低成本的存储服务。
