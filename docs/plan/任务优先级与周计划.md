# MostBox 任务优先级与当前执行计划

> 本文是当前任务板。MVP 候选已收尾，当前只保留小范围 Alpha 需要推进的任务；已完成内容压缩为快照，避免任务板继续堆旧账。

## 一、当前目标

目标：进入小范围 Alpha，让真实用户验证“发链接 + 在线做种 + 下载者接力”的理解成本和稳定性。

成功标准：

1. 找到 5 个真实下载者完成一次端到端下载。
2. 保持 2 台机器长期在线做种至少 48 小时。
3. 记录首次使用中的 Top 阻塞问题，并按影响程度修复。
4. 用户能理解 MostBox 不是云盘；文件可用性来自当前在线种子。
5. 管理台能帮助解释“为什么现在下载不到”。

## 五、运营验证问题

| 阶段         | 只问一个问题                                   |
| ------------ | ---------------------------------------------- |
| MVP 候选     | 用户是否理解“发链接 + 在线做种 + 下载者接力”？ |
| 小范围 Alpha | 用户是否愿意让下载完成的文件继续做种？         |
| v1 前        | 管理台是否能解释“为什么现在下载不到”？         |

统一定位文案：

> MostBox 是一个 P2P 文件分享工具：你发布文件得到 `most://` 链接，别人下载后默认继续做种，文件靠在线用户传播。

如果用户问“这是不是备份/云盘”，回答必须一致：

> 不是。重要数据请自己保存；MostBox 只负责 P2P 分享和校验，不承诺永久保存。
