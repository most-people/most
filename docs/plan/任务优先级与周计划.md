# MostBox 任务优先级与周计划

> 目标：把现有 P2P 文件能力收敛成类 BitTorrent 的分享工具。每一关都有主线任务、验收条件和解锁下一关的标准。

---

## 一、优先级规则

| 优先级 | 判断标准 | 示例 |
| ------ | -------- | ---- |
| P0 | 没有它，MVP 闭环跑不通 | CID/Merkle、`most://`、topic 下载、下载后做种 |
| P1 | MVP 可用但不够顺手 | 管理台、seed policy、暂停/恢复、日志 |
| P2 | 提升体验或增长验证 | 种子健康度、Quick Start、早期用户访谈 |
| P3 | 后续再评估 | `magnet:` 兼容、浏览器长期做种、内置加密 |

当前主线：

```
现有 P2P 文件分享
  → 稳定 CID + Merkle 校验
  → 保留 most:// 分享链接
  → 下载后默认做种
  → 做种持久化和管理台
  → 小范围 Alpha
```

## 二、现有功能可复用清单

| 已有能力 | 用法 | 下一步 |
| -------- | ---- | ------ |

| Web3 工具箱 | 独立工具 | 保留入口，不接入分享主流程 |

## 三、8 周主线地图

### [x] 第 1 关：协议基线与测试护栏

目标：把文件身份和完整性校验钉死。

| 子任务 | 验收 |
| ------ | ---- |
| CID 黄金样本测试 | 空文件、小文件、跨 chunk 文件 CID 固定 |
| Merkle chunk 工具 | root/proof 单测覆盖 |
| 上传元数据 | 发布后得到 CID + `chunkMerkleRoot` |
| 下载双重校验 | 篡改文件时 CID 或 Merkle 校验失败 |

### [x] 第 2 关：无链 P2P 做种闭环

目标：发布者退出后，下载者种子仍能继续提供下载。

| 子任务 | 验收 |
| ------ | ---- |
| 节点持有记录 | 重启后可读取 |
| 按 CID digest join topic | 启动后 join 已持有 CID |
| 从 topic 拉取完整副本 | 双节点本地拉取成功 |
| 下载后默认做种 | 第三个节点可从下载者拉取 |
| 3 节点集成测试 | A 发布，B 下载做种，A 退出，C 从 B 下载 |

### [ ] 第 3 关：做种服务管理台

目标：让普通机器和服务器都能作为做种服务运行。

| 子任务 | 验收 |
| ------ | ---- |
| seed policy 字段 | 无订单、USDT、报价字段 |
| Web 管理台首页 | 在线状态、容量、节点 ID、监听地址 |
| 持有副本列表 | CID、size、root、topic、join 状态 |
| 日志 | join、pull、verify、serve、error |
| OpenAPI | 状态、配置、策略、holdings、日志 |

解锁条件：

- 管理台不再出现云端订单、接单、报价、USDT、质押、赏金等文案。
- Ubuntu / Windows 本机均可作为无头做种服务启动。

### [ ] 第 4 关：用户体验收敛

目标：把“发布链接”和“输入链接下载”做得像普通分享工具。

| 子任务 | 验收 |
| ------ | ---- |
| 主应用删除云端入口 | 只保留本地、收藏、回收站、节点管理 |
| 分享弹窗改文案 | 强调本机在线与下载者接力做种 |
| 下载失败提示 | 无 peer、校验失败、被取消可读 |
| Web3 保留 | `/web3` 独立入口可用 |

### [ ] 第 5-6 关：Alpha 打磨

目标：真实文件、真实网络、小范围验证。

| 子任务 | 验收 |
| ------ | ---- |
| Quick Start | 发布者/下载者/daemon 三条路径 |
| 100MB / 1GB 长测 | 下载后双重校验通过 |
| 重启恢复 | holding 自动 join |
| 暂停/恢复做种 | 单文件控制可用 |
| 种子健康度 | 至少展示 peer 数或最近连接 |

### [ ] 第 7-8 关：小范围 Alpha

目标：拿真实反馈校正产品。

| 子任务 | 验收 |
| ------ | ---- |
| 招募 5 个下载者 | 至少 10 次完整下载 |
| 招募 2 个长期做种机器 | 至少 48 小时在线 |
| 收集首次使用问题 | 归类到产品/技术/文案 |
| 修复 Top 阻塞问题 | Top 5 关闭 |

## 四、运营验证

每周只问一个关键问题：

| 阶段 | 问题 |
| ---- | ---- |
| 第 1-2 周 | 用户是否理解“发链接 + 在线做种”？ |
| 第 3-4 周 | 用户是否愿意让下载完成的文件继续做种？ |
| 第 5-6 周 | 管理台是否能解释“为什么现在下载不到”？ |
| 第 7-8 周 | 真实用户是否会把它当成网盘误用？ |

早期定位文案：

> MostBox 是一个 P2P 文件分享工具：你发布文件得到 `most://` 链接，别人下载后默认继续做种，文件靠在线用户传播。

如果用户问“这是不是备份/云盘”，回答必须一致：

> 不是。重要数据请自己保存；MostBox 只负责 P2P 分享和校验，不承诺永久保存。
