# MostBox MVP 落地计划

> 本文用于把新方向收敛成可执行 MVP。原则：先验证“发布者在线时能把文件传播出去，下载者自动成为种子”。

---

## 一、MVP 定义

MVP 只做一个最小闭环：

```
发布者选择文件
  → 生成 CID 与 chunkMerkleRoot
  → 生成 most:// 分享链接
  → 发布者做种
  → 下载者凭链接下载并校验
  → 下载者默认持续做种
  → 发布者退出后，新下载者可从下载者种子继续下载
```

成功标准：

1. 只凭 `most://` 链接能完成发现、下载、校验。
2. 下载完成后自动加入做种列表，重启后自动重新 join topic。
3. 发布者退出后，只要至少一个下载者仍在线做种，文件仍可继续传播。

## 二、MVP 范围

### 必须包含

| 模块 | MVP 要求 | 验收方式 |
| ---- | -------- | -------- |
| CID 与 Merkle | 复用 UnixFS CID v1；按 256KB chunk 生成 `chunkMerkleRoot` | 黄金样本测试 + 篡改检测 |
| 分享链接 | 保留 `most://<cid>?filename=...&r=...` | 链接解析单测 |
| P2P 下载 | topic 使用 `cid.multihash.digest` | 本地多节点集成测试 |
| 下载后做种 | 下载完成后写入 Hyperdrive 并记录 holding | 第三个节点可从下载者拉取 |
| 做种持久化 | `node-holdings.json` 保存 CID、size、root、topic、path | 重启后自动 join |
| 管理台 | 展示在线状态、容量、持有副本、日志、seed policy | 手动验证 |
| seed policy | 自动做种开关、容量、单文件上限、并发数、上传限速 | API 测试 |

### 明确不做

| 能力 | MVP 决策 |
| ---- | -------- |
| 上链存储协议 | 不做 |
| 智能合约 / USDT / 质押 / 结算 | 不做 |
| 存储订单 / 报价 / 手动选节点 | 不做 |
| 赏金猎人 / fraud proof / Treasury | 不做 |
| 云端上传或托管 | 不做 |
| 默认加密 | 不做；用户自行本地加密 |
| 浏览器长期做种 | 不承诺；正式能力在桌面端或 daemon |
| `magnet:` 兼容 | 暂不做；保留 `most://` |

Web3 工具箱保留，但不作为发布、下载或做种的前置条件。

## 三、落地切片

### MVP-0：协议与校验

| 任务 | 产出 | 验证 |
| ---- | ---- | ---- |
| CID 黄金样本 | 固定 `ipfs-unixfs-importer` 输出 | `npm run test:protocol` |
| Merkle 工具 | `chunkMerkleRoot` 与 proof 工具 | 单元测试 |
| 链接解析 | `most://` 必须含 `r` | 单元测试 |
| 下载校验 | CID + Merkle 双重校验 | 篡改测试 |

### MVP-1：P2P 分享闭环

| 任务 | 产出 | 验证 |
| ---- | ---- | ---- |
| 发布后做种 | 发布文件自动 join CID topic | 本机可被发现 |
| 下载后做种 | 下载完成自动写入 holding | 新下载者可从下载者拉取 |
| 持有记录 | `node-holdings.json` | 重启后自动 join |
| 失败提示 | 无 peer、校验失败、节点退出 | API 返回明确错误 |

### MVP-2：做种服务体验

| 任务 | 产出 | 验证 |
| ---- | ---- | ---- |
| Web 管理台 | 状态、容量、持有副本、日志 | 手动测试 |
| seed policy | 自动做种、文件上限、并发、限速 | API 测试 |
| 桌面端接入 | 主应用分享/下载文案统一 | lint + 手动测试 |
| Quick Start | 发布者、下载者、daemon 三条路径 | 新用户可照做 |

## 四、用户故事

### 发布者

1. 选择文件。
2. 看到 CID、`chunkMerkleRoot` 和 `most://` 链接。
3. 复制链接发给别人。
4. 保持应用在线，让第一批下载者拉到文件。

验收：第一个下载者能凭链接下载并校验。

### 下载者

1. 输入 `most://` 链接。
2. 客户端 join topic 并下载。
3. 下载完成后显示校验结果。
4. 默认继续做种。

验收：第二个下载者能从第一个下载者处拉取文件。

### 做种节点 / daemon

1. 配置数据目录、容量和做种策略。
2. 查看持有副本和 topic join 状态。
3. 后台长期在线，持续为本机持有的 CID 做种。

验收：重启后自动恢复做种列表。

## 五、MVP 指标

| 指标 | 目标 |
| ---- | ---- |
| 本地 3 节点传播成功率 | 100% |
| 100MB 文件下载后校验 | CID 与 `chunkMerkleRoot` 均一致 |
| 发布者退出后下载 | 至少 1 个下载者种子在线即可成功 |
| 节点重启恢复 | 自动 join 已持有 CID topic |
| seed policy API | 不包含订单、USDT、报价字段 |

## 六、完成定义

MVP 完成时，应能现场演示：

1. A 发布文件并复制 `most://` 链接。
2. B 输入链接下载，校验通过后自动做种。
3. A 退出。
4. C 输入同一链接，从 B 下载成功并通过双重校验。
5. B 重启后，管理台仍能看到该 CID 并自动 join topic。
