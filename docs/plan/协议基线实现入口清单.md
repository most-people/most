# 协议基线实现入口清单

> 梳理 `server/src/core/cid.js`、发布/下载链路、做种持有记录和数据目录写入点。

## CID 入口

| 入口 | 位置 | 现状 |
| ---- | ---- | ---- |
| `calculateCid(content, options)` | `server/src/core/cid.js` | 使用 `ipfs-unixfs-importer` 生成 UnixFS CID v1，默认 `rawLeaves: true`、`cidVersion: 1`、`wrapWithDirectory: false` |
| `validateCidString(cidString)` | `server/src/core/cid.js` | 校验非空字符串与 CID v1 `b` 前缀 |
| `parseMostLink(link)` | `server/src/core/cid.js` | 接受 `most://<cid>?filename=...&r=...`，其中 `r` 为必填 `chunkMerkleRoot` |

## 发布链路

| 步骤 | 入口 | 写入/输出 |
| ---- | ---- | --------- |
| 接收文件 | `POST /api/publish` | multipart 临时落盘，发布后删除临时文件 |
| 发布文件 | `engine.publishFile(filePath, filename)` | 计算 CID 与 `chunkMerkleRoot` |
| 写入 Hyperdrive | `drive.createWriteStream('/' + cid)` | Hyperdrive key 固定为 `/<cid>` |
| 写入文件元数据 | `published-files.json` | 用户可见文件名、CID、Merkle root、收藏和目录信息 |
| 写入做种记录 | `node-holdings.json` | CID、size、root、topic、local path、join 状态 |
| 返回客户端 | `publishResult` | `cid`、`most://` 链接、`chunkMerkleRoot` |

## 下载链路

| 步骤 | 入口 | 写入/输出 |
| ---- | ---- | --------- |
| 请求下载 | `POST /api/download` | 解析 `most://`，异步启动下载任务 |
| join topic | `engine.downloadFile(link, taskId)` | topic = `cid.multihash.digest` |
| 拉取内容 | `#waitForDriveContent` | 从任意可用 peer 拉取完整文件 |
| 写入本地文件 | `fs.createWriteStream(savePath)` | 保存到本机数据目录 |
| 双重校验 | `calculateCid` + `calculateChunkMerkleRoot` | CID 与 `r` 都必须匹配 |
| 下载后做种 | 写入 Hyperdrive + holding | 下载者成为新的种子 |

## 做种持有记录

| 入口 | 说明 |
| ---- | ---- |
| `GET /api/node/holdings` | 列出当前本机做种副本 |
| `POST /api/node/holdings` | 添加持有记录并 join topic |
| `engine.pullByCid(input)` | 按 CID 和 root 拉取完整副本 |
| `engine.joinCidTopic(cid)` | 按 CID digest 加入文件 topic |

holding 记录必须和 `published-files.json` 分开，避免做种副本污染用户文件管理视图。

## 数据目录写入点

| 路径 | 写入入口 | 说明 |
| ---- | -------- | ---- |
| `~/.most-box/config.json` | `createNodeConfigStore` | 保存数据目录和 seed policy |
| `dataPath/published-files.json` | `#savePublishedMetadata()` | 用户可见文件列表 |
| `dataPath/node-holdings.json` | `#saveHoldingsMetadata()` | 做种副本列表 |
| `dataPath/trash-files.json` | `#saveTrashMetadata()` | 回收站元数据 |
| `dataPath/channels.json` | `#saveChannelsMetadata()` | 聊天频道列表 |
| `dataPath/<sanitizedFileName>` | `downloadFile()` | 下载完成后的本地文件副本 |
| `dataPath` 内 Corestore 文件 | Corestore / Hyperdrive | P2P 内容存储 |
| 系统临时目录 `mostbox-uploads` | `parseMultipartBusboy` | 发布中转文件，发布后删除 |

## 验证边界

- CID 黄金样本测试直接覆盖 `calculateCid`。
- Merkle 工具先独立单测，再由发布/下载链路调用。
- `most://` 链接必须包含 CID、filename 和 `r`。
- P2P pull 必须要求 `chunkMerkleRoot`，不能只凭 CID 保存文件。
