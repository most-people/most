# MostBox 实施路线图

> 路线图按里程碑推进，不再按旧周计划推进。主线保持 P2P 分享与默认做种，不回到上链订单、支付、质押、赏金或云端上传。

## 里程碑总览

| 里程碑          | 目标                                          | 状态     | 通过门槛                                       |
| --------------- | --------------------------------------------- | -------- | ---------------------------------------------- |
| M0 产品收敛     | 从付费存储市场收敛为 P2P 分享工具             | 已完成   | 文档和 UI 不再承诺云盘、订单或付费保种         |
| M1 协议基线     | CID、`most://`、下载校验稳定                  | 已完成   | `npm run test:protocol` 覆盖核心路径           |
| M2 P2P 接力     | 下载者自动成为种子，发布者退出后仍可传播      | 已完成   | 三节点接力测试通过                             |
| M3 MVP 候选     | 管理台、daemon、主应用体验和 Quick Start 收尾 | 已完成   | 新用户能按文档完成端到端演示                   |
| M4 小范围 Alpha | 真实机器、真实文件、真实反馈                  | 当前阶段 | 5 个下载者、2 台长期做种机器、Top 阻塞问题关闭 |
| M5 v1 打磨      | 健康度、诊断、打包、体验增强                  | Alpha 后 | 用户能稳定解释和处理下载失败                   |

## M3：MVP 候选

目标：把已经完成的协议和 P2P 能力包装成可试用产品。

| 任务           | 产出                                    | 验证                   |
| -------------- | --------------------------------------- | ---------------------- |
| Quick Start    | 发布者、下载者、daemon 三条路径         | 新用户照做             |
| 主应用文案收敛 | 分享、下载、失败提示都使用 P2P 做种口径 | 手动检查               |
| 管理台收尾     | 状态、容量、holding、日志、设置清晰     | 手动验证               |
| 安全门槛       | 本机监听或公网访问口令/警告策略         | 手动验证               |
| 验收脚本       | A 发布、B 下载做种、A 退出、C 下载      | 本地和真实网络各跑一次 |
| 工具箱入口隔离 | 笔记、Web3、聊天不成为文件分享前置条件  | 手动检查               |

里程碑：v0.4.x 可以作为 MVP 候选发给小范围用户。

本轮已完成 MVP 候选收尾和 Alpha 前长测：Quick Start、主应用文案、管理台安全提示、端到端接力、本地重启恢复、100MB / 1GB 文件测试均已通过。

## M4：小范围 Alpha

目标：让真实用户验证“发链接 + 在线做种”的理解成本和稳定性。

| 任务             | 产出                                 | 验证             |
| ---------------- | ------------------------------------ | ---------------- |
| 100MB / 1GB 长测 | 下载、校验、做种、重启恢复记录       | 手动长测         |
| 早期用户招募     | 5 个下载者，2 台长期在线做种机器     | 至少 48 小时观察 |
| 问题归类         | 产品理解、网络失败、文案、性能、安装 | 反馈表           |
| Top 阻塞修复     | 关闭最影响首次使用的 5 个问题        | 回归验证         |

里程碑：v0.5.0 小范围 Alpha。

## M5：v1 打磨

目标：提升诊断能力和长期使用体验，但不改变 MVP 协议。

| 能力               | 处理方式                                |
| ------------------ | --------------------------------------- |
| 种子健康度         | 展示 peer 数、最近连接、最后 serve 时间 |
| 下载同名冲突       | 给出覆盖、另存、取消等明确选择          |
| 分享历史           | 保存本机发布过的链接和 CID              |
| 单文件暂停/恢复    | 保持默认自动做种，只提供单文件级管理    |
| 导入/导出 holdings | 便于迁移 daemon 数据目录                |
| 桌面打包体验       | Windows / macOS / Linux 安装和自动启动  |

## 长期评估，不进入近期主线

| 能力                              | 当前决策                            |
| --------------------------------- | ----------------------------------- |
| `magnet:` 双格式                  | 后续看用户需求，单独设计兼容层      |
| 浏览器长期做种                    | 不承诺；正式做种依赖桌面端或 daemon |
| 默认应用层加密                    | 不内置；用户先本地加密再分享密文    |
| 上链存储协议                      | 不实现                              |
| 智能合约 / USDT / 质押 / 结算     | 不实现                              |
| 订单、报价、手动选节点            | 不实现                              |
| 赏金猎人 / fraud proof / Treasury | 不实现                              |
| 云端上传或托管                    | 不实现                              |

笔记、Web3 和 P2P 聊天作为独立产品入口保留，不影响文件分享路线。

## 独立工具箱维护轨道

这条轨道不阻塞 MostBox 文件分享 MVP，但需要随代码变化保持计划同步。

| 工具     | 当前状态                                                                                      | 下一步                                               |
| -------- | --------------------------------------------------------------------------------------------- | ---------------------------------------------------- |
| 笔记     | 基础闭环已成型：Markdown 编辑、本地保存、文件夹、搜索、移动、私密笔记、加密备份和本地导入导出 | 补齐边界文案、冲突/失败状态验收、备份 API 可用性观察 |
| Web3     | 独立账户和密钥工具，可供笔记加密复用                                                          | 保持独立入口，不成为文件分享前置条件                 |
| P2P 聊天 | 频道、消息、peer 状态和 WebSocket 订阅基础完成                                                | 后续按真实使用反馈排优先级                           |

独立工具可以使用 Web3 密钥或 P2P 底座，但不能改变 MostBox 文件分享协议的核心不变量。
