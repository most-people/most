# MostBox 实施路线图

> 新主线：P2P 分享与默认做种。路线图不再包含上链订单、支付、质押、赏金猎人或云端上传。

---

## MVP 优先级

| 切片 | 目标 | 是否首版必做 |
| ---- | ---- | ------------ |
| MVP-0 | CID + `most://` 链接稳定 | 是 |
| MVP-1 | 发布、下载、下载后做种闭环 | 是 |
| MVP-2 | 做种 daemon / 管理台 / 节点设置 | 是 |
| v1 扩展 | 种子健康度、限速、暂停、导入导出、打包体验 | MVP 后推进 |

## Phase 1 — 协议基线

目标：跑通 `CID → most:// → 下载校验`。

| 任务 | 产出 | 验证 |
| ---- | ---- | ---- |
| CID 计算对齐 | UnixFS CID v1 黄金样本 | 黄金样本测试 |
| 链接解析 | CID 与 filename 必填 | 单元测试 |
| 下载完整性校验 | 下载后重算 CID | 篡改测试 |

里程碑：v0.2.0 可稳定生成和校验 MostBox 分享链接。

## Phase 2 — P2P 传播闭环

目标：发布者在线时把文件传给第一批下载者，下载者自动接力做种。

| 任务 | 产出 | 验证 |
| ---- | ---- | ---- |
| 发布后 join topic | 发布者成为种子 | 双节点下载 |
| 下载后写入 Hyperdrive | 下载者成为种子 | 三节点接力下载 |
| holding 持久化 | `node-holdings.json` | 重启恢复 |
| 无 peer/校验失败处理 | 清晰错误码和日志 | 集成测试 |

里程碑：v0.3.0 可演示 A 发布、B 下载做种、A 退出、C 从 B 下载。

## Phase 3 — 做种服务与管理台

目标：让桌面端和后台 daemon 都能作为长期在线的做种服务。

| 任务 | 产出 | 验证 |
| ---- | ---- | ---- |
| 节点设置 | 容量、文件上限 | API 测试 |
| Web 管理台 | 状态、容量、持有副本、日志 | 手动测试 |
| OpenAPI | 状态、配置、策略、holdings、日志 | API 文档检查 |
| 桌面端控制台 | 主应用与管理台入口清晰 | lint + 手动测试 |

里程碑：v0.4.0 用户能查看并管理本机正在做种的文件。

## Phase 4 — Alpha 打磨

目标：让小范围用户真实使用。

| 任务 | 产出 | 验证 |
| ---- | ---- | ---- |
| Quick Start | 发布者、下载者、daemon 三条路径 | 新用户照做 |
| 种子健康度 | peer 数、最近连接、最后做种时间 | 手动测试 |
| 暂停/恢复做种 | 单文件控制 | 集成测试 |
| 长测 | 100MB / 1GB 文件传播 | 手动长测 |

里程碑：v0.5.0 小范围 Alpha。

## 暂不进入主线的能力

| 能力 | 处理方式 |
| ---- | -------- |
| 上链存储协议 | 不做 |
| 智能合约 / USDT / 质押 / 结算 | 不做 |
| 订单、报价、手动选节点 | 不做 |
| 赏金猎人 / fraud proof / Treasury | 不做 |
| 云端上传或托管 | 不做 |
| `magnet:` 双格式 | 后续单独评估 |
| 默认应用层加密 | 用户可自行本地加密 |
| 浏览器长期做种 | 不承诺；正式能力在桌面端或 daemon |

Web3 工具箱作为独立产品入口保留，不影响文件分享路线。
