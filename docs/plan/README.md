# MostBox 计划文档索引

`docs/plan` 下的文档分为三层：先看 MVP 怎么落地，再看完整 v1 路线，最后按专题查细节。

## 推荐阅读顺序

1. `任务优先级与周计划.md`：从第 1 天开始做什么，每周如何验收。
2. `MVP落地计划.md`：第一版到底做什么、不做什么、如何验收。
3. `实施路线图.md`：MVP 之后如何扩展到完整 v1。
4. `项目计划书.md`：完整愿景、市场定位、经济模型和风险边界。
5. `技术架构.md`：现有代码基础、待实现模块和非 MVP 能力。
6. `智能合约.md`：订单、节点注册、质押、审计和结算的合约设计。
7. `零知识证明.md`：为什么 MVP 不做 ZK/PoSt，以及 Merkle audit 方案。
8. `节点运营.md`：节点如何部署、报价、存储、结算和应对审计。
9. `加密与存储.md`：公共内容保种与可选本地加密边界。

## 当前 MVP 口径

首版只验证一个闭环：

```
CID + Merkle 校验
  → Hyperswarm 完整副本保种
  → 测试网/本地 USDT 订单
  → 用户手动选择节点
  → 节点 pullConfirm
  → 下载者凭 CID 下载并校验
  → 最小随机 chunk 审计
```

节点入口采用 API 优先：Web 管理台给人用，OpenAPI 给 AI/脚本用，薄 CLI 只负责安装、启动和诊断。以下能力不阻塞 MVP：原生代币、DAO、多稳定币、默认加密、浏览器常驻节点、完整 CLI 产品、自动抽签、自动 repair、赏金猎人市场化运营、ZK/PoSt、复杂 Treasury 分账。
