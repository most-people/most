# MostBox 计划文档索引

`docs/plan` 是 MostBox 当前产品和工程计划的单一入口。当前仓库的产品壳是 MOST PEOPLE 去中心化工具箱；其中 MostBox 文件分享仍是 MVP 主线，目标是把它收敛成类 BitTorrent / 磁力链接的 P2P 文件分享与做种工具。

笔记、Web3、P2P 聊天等入口作为独立工具存在。它们可以复用登录、加密或 P2P 底座，但不改变 MostBox 文件分享的协议、验收口径或“不承诺云盘/备份”的产品边界。

## 当前成功标准

MVP 必须能现场演示这个闭环：

```
A 发布文件并得到 most:// 链接
  -> B 凭链接下载并通过 CID 校验
  -> B 下载完成后默认继续做种
  -> A 退出
  -> C 仍能从 B 下载同一文件并通过 CID 校验
```

计划文档只追踪仍需推进的 Alpha 验证和后续 v1 能力，不保留已完成任务清单或旧周复盘。

## 文档分工

| 文档                      | 职责                                             | 维护规则                                     |
| ------------------------- | ------------------------------------------------ | -------------------------------------------- |
| `任务优先级与周计划.md`   | 当前任务板、优先级、验收顺序                     | 只写接下来要做的任务                         |
| `MVP落地计划.md`          | MVP 范围、非目标、最终验收剧本                   | 范围变化先改这里                             |
| `实施路线图.md`           | MVP 到 Alpha 到 v1 的阶段门槛                    | 不写日常任务细节                             |
| `项目计划书.md`           | 定位、价值、风险和竞品关系                       | 保持产品叙事稳定                             |
| `技术架构.md`             | 分层、核心模块、现有实现和后置能力               | 和代码入口保持一致                           |
| `协议基线实现入口清单.md` | CID、发布、下载、holding、数据目录入口           | 改协议或存储路径时同步                       |
| `独立工具箱与笔记.md`     | 笔记、Web3、P2P 聊天等非文件分享主线的边界和状态 | 改笔记数据模型、备份、加密或工具箱入口时同步 |
| `节点运营.md`             | daemon、管理台、配置、安全和排障                 | 改节点 API 或管理台时同步                    |
| `加密与存储.md`           | 默认公开分享、私密文件自加密                     | 改隐私模型时同步                             |

## 不再进入主线

以下能力不属于 MostBox 文件分享主线：上链存储协议、智能合约、USDT 支付、质押、订单、赏金猎人、fraud proof、Treasury、云端下单、云端上传或付费保种市场。

笔记云备份、Web3 产品入口和以太坊钱包工具保留为独立工具能力，不作为 MostBox 文件发布、下载或做种的前置条件。
