# MostBox 做种服务规格

> “节点”指本机或服务器上的做种服务，不是接单矿工，也不承担付费存储承诺。

## 一、做种角色

MostBox 做种服务负责保存本机已发布或已下载的完整文件副本，并在应用在线时向其他 peer 提供上传带宽。

| 形态 | 适合对象 | MVP 定位 |
| ---- | -------- | -------- |
| 桌面端做种 | 普通用户、家用电脑 | 发布、下载、默认做种 |
| 后台 daemon | NAS、Ubuntu 服务器、长期在线机器 | 稳定做种、管理台观察 |

核心职责：

- 发布成功后自动 join CID topic 并做种。
- 下载成功后自动写入 Hyperdrive，加入 holding，并继续做种。
- 重启后读取 `node-holdings.json`，后台分批重新 join 已持有 CID topic。
- 提供 HTTP API、WebSocket 事件、OpenAPI 和 Web 管理台。
- 按本地节点设置控制数据目录、容量和单文件大小。

不做订单、报价、质押、收益、赔付或审计奖励。

## 二、做种流程

### 2.1 发布后做种

```
发布文件
  -> 计算 UnixFS CID v1
  -> 写入 Hyperdrive /<cid>
  -> 写入 published-files.json
  -> 写入 node-holdings.json
  -> join topic = cid.multihash.digest
  -> 生成 most:// 链接
```

### 2.2 下载后做种

```
输入 most:// 链接
  -> join CID topic
  -> 拉取 Hyperdrive 中精确的 /<cid> 文件
  -> 重算 UnixFS CID v1
  -> CID 校验通过
  -> 保存到本机下载目录
  -> 写入本机 Hyperdrive /<cid>
  -> 写入 node-holdings.json
  -> 继续 serve 给其他 peer
```

### 2.3 重启恢复

```
daemon 启动
  -> 读取 node-holdings.json
  -> 校验 CID、size、topic 字段
  -> 将 CID 标记为 queued
  -> 后台分批 join topic
  -> 管理台展示 queued / joining / active / paused / error
```

## 三、节点设置

| 字段 | 说明 |
| ---- | ---- |
| `dataPath` | 数据目录，保存 Corestore、元数据、下载目录等 |
| `capacityBytes` | 本机做种空间上限 |
| `maxFileSizeBytes` | 单文件大小上限 |

发布和下载成功后固定自动做种。MostBox 不提供全局关闭自动做种开关、产品层同时做种上限或应用层限速。

## 四、管理台范围

| 页面 | MVP 功能 | 状态 |
| ---- | -------- | ---- |
| 概览 | 在线状态、版本、节点 ID、监听地址、容量 | 基础完成 |
| 存储 | 已持有 CID、文件大小、做种状态 | 基础完成 |
| 设置 | 数据目录、容量上限、单文件上限 | 基础完成 |
| 日志 | join、pull、verify、serve、error | 基础完成，继续打磨可读性 |

Alpha 前必须完成管理台安全门槛：默认仅本机访问，或公网暴露时要求访问口令并给出明确安全提示。MVP 不做复杂 RBAC。

## 五、API 与自动化入口

| 入口 | 范围 |
| ---- | ---- |
| `GET /api/node/status` | 节点状态、容量、网络、holding |
| `GET /api/node/config` | 读取数据目录、容量、单文件上限 |
| `POST /api/node/config` | 更新节点配置 |
| `GET /api/node/policy` | 读取本地文件大小策略 |
| `POST /api/node/policy` | 更新本地文件大小策略 |
| `POST /api/node/policy/evaluate` | 评估文件是否符合节点限制 |
| `GET /api/node/holdings` | 列出本机持有的可做种副本 |
| `POST /api/node/holdings` | 手动添加 holding 并 join topic |
| `GET /api/node/logs` | 读取最近节点日志 |
| `DELETE /api/node/logs` | 清空节点日志 |
| `POST /api/p2p/pull` | 按 CID 或 `most://` 拉取完整副本 |
| `GET /api/openapi.json` | OpenAPI 文档 |
| WebSocket | 节点状态、下载/做种进度、错误事件 |

薄 CLI 后续只做初始化、安装服务、启动/停止和诊断，不承载产品主流程。

## 六、P2P 安全边界

| 风险 | 应对 |
| ---- | ---- |
| 恶意 peer 返回坏数据 | 下载后重算 CID 校验 |
| 共享写入 Hyperdrive 历史被污染 | 只接受精确 `/<cid>` 文件并重算 CID；后续评估 per-peer drive |
| 无种子 | UI 明确提示需要至少一个 peer 在线 |
| 链接泄露 | 设计如此；私密文件先本地加密 |
| 上传带宽占满 | 暂不做应用层限速，依赖系统、Hypercore 和网络栈自然限制兜底 |
| 本地空间占满 | capacity 限制与做种列表管理 |
