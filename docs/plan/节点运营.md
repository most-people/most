# MostBox 做种服务规格

> “节点”在新路线中指本机或服务器上的做种服务，不是接单矿工，也不承担付费存储承诺。

---

## 一、做种角色

MostBox 做种服务负责保存本机已发布/已下载的完整文件副本，并在应用在线时向其他 peer 提供上传带宽。

| 形态 | 适合对象 | MVP 定位 |
| ---- | -------- | -------- |
| 桌面端做种 | 普通用户、家用电脑 | 发布、下载、默认做种 |
| 后台 daemon | NAS、Ubuntu 服务器、长期在线机器 | 稳定做种、管理台观察 |

### 核心职责

- 发布成功后自动 join CID topic 并做种。
- 下载成功后自动写入 Hyperdrive，加入 holding，并继续做种。
- 重启后读取 `node-holdings.json`，后台分批重新 join 已持有 CID topic。
- 提供 HTTP API、WebSocket 事件、OpenAPI 和 Web 管理台。
- 按本地节点设置控制容量和单文件大小。

不做订单、报价、质押、收益、赔付或审计奖励。

## 二、做种流程

### 2.1 发布后做种

```
发布文件
  → 计算 CID
  → 写入 Hyperdrive /<cid>
  → 写入 holding
  → join topic = cid.multihash.digest
  → 生成 most:// 链接
```

### 2.2 下载后做种

```
输入 most:// 链接
  → join topic
  → 拉取完整文件
  → CID 校验通过
  → 写入 Hyperdrive /<cid>
  → 写入 holding
  → 继续 serve 给其他 peer
```

### 2.3 重启恢复

```
daemon 启动
  → 读取 node-holdings.json
  → 验证字段
  → 将 CID 标记为 queued
  → 后台分批 join topic
  → 管理台展示 queued / joining / active / paused / error
```

## 三、节点设置

默认配置：

| 字段 | 默认 | 说明 |
| ---- | ---- | ---- |
| `capacityBytes` | 100 GiB | 本机做种空间上限 |
| `maxFileSizeBytes` | 全局上传上限 | 单文件大小上限 |

发布和下载成功后固定自动做种；MostBox 不提供关闭开关、产品层同时做种上限或应用层限速。

## 四、Web 管理台范围

| 页面 | MVP 功能 |
| ---- | -------- |
| 概览 | 在线状态、版本、节点 ID、监听地址、已用/可用容量 |
| 存储 | 已持有 CID、文件大小、做种状态 |
| 设置 | 数据目录、容量上限、单文件上限 |
| 日志 | join、pull、verify、serve、error |

Web 管理台必须默认只监听本机或要求设置访问口令；服务器公网暴露时需要明确安全提示。MVP 不做复杂 RBAC。

## 五、API 与 AI 入口

| 入口 | 范围 |
| ---- | ---- |
| HTTP API | 配置、状态、节点设置、holdings、日志 |
| WebSocket | 节点状态、下载/做种进度、错误事件 |
| OpenAPI 文档 | 给脚本和第三方集成使用 |
| 薄 CLI | 初始化、安装服务、启动/停止、打印诊断信息 |

## 六、P2P 安全

| 风险 | 应对 |
| ---- | ---- |
| 恶意 peer 返回坏数据 | 下载后重算 CID 校验 |
| 共享写入 Hyperdrive 历史被污染 | 只接受精确 `/<cid>` 文件并重算 CID；后续评估 per-peer announce / per-peer drive |
| 无种子 | UI 明确提示需要至少一个 peer 在线 |
| 链接泄露 | 设计如此；私密文件先本地加密 |
| 上传带宽占满 | 暂不做应用层限速，依赖系统、Hypercore 和网络栈自然限制兜底 |
| 本地空间占满 | capacity 限制与做种列表管理 |
