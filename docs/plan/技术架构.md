# MostBox 技术架构

> 以 `项目计划书.md` 为准：v1 是 USDT 单资产、用户自选副本数的公共内容保种市场，支持竞价、续费和审计；不发行原生代币。

---

## 一、分层架构

```
┌─────────────────────────────────────────────────────────────┐
│                      用户交互层                              │
│  Web 管理台  │  Electron 桌面端  │  薄 CLI  │  AI/MCP(未来) │
├─────────────────────────────────────────────────────────────┤
│                      接口自动化层                            │
│  HTTP API  │  WebSocket 事件  │  OpenAPI 文档  │  错误码       │
├─────────────────────────────────────────────────────────────┤
│                      应用服务层                              │
│  文件管理  │  聊天频道  │  节点运营  │  订单管理  │  钱包    │
├─────────────────────────────────────────────────────────────┤
│                      内容完整性层                            │
│  UnixFS CID v1  │  256KB chunk  │  chunkMerkleRoot  │  本地校验  │
├─────────────────────────────────────────────────────────────┤
│                      P2P 网络层                              │
│  Hyperswarm 4.x  │  Hyperdrive 13.x  │  Hypercore 17.x     │
├─────────────────────────────────────────────────────────────┤
│                      区块链层                                │
│  USDT 支付  │  USDT 质押  │  OrderBook  │  FraudProof     │
└─────────────────────────────────────────────────────────────┘
```

## 二、技术栈

| 层级   | 技术                             | 作用                                   |
| ------ | -------------------------------- | -------------------------------------- |
| 网络层 | Hyperswarm 4.x                   | 节点发现、NAT 打洞、P2P 组网           |
| 存储层 | Hyperdrive 13.x + Hypercore 17.x | 完整文件副本存储、内容寻址             |
| 校验层 | CID v1 + Merkle Tree             | 文件身份、chunk 完整性、链上验证锚     |
| 合约层 | Base (Solidity)                  | USDT 订单、质押、随机抽查/赏金猎人审计、结算、手动修复 |
| 接口层 | Hono API + WebSocket + OpenAPI   | Web 管理台、桌面端、薄 CLI、AI Agent 共用 |
| 交互层 | Next.js 16 + React 19            | 浏览器 demo、节点 Web 管理台、桌面客户端 |
| 部署层 | daemon + Docker + systemd/Windows Service | 服务器节点长期运行                     |

### CID 生成参数

MostBox v1 复用当前 `server/src/core/cid.js` 实现：`ipfs-unixfs-importer@16.1.5`、`cidVersion: 1`、`rawLeaves: true`、`wrapWithDirectory: false`。未显式写出的 importer 默认参数由 `package-lock.json` 锁定，升级依赖前必须通过 CID 黄金样本测试。

### 接口优先原则

节点能力先沉到 HTTP API 与 WebSocket 事件层，再由不同入口复用：

| 入口 | 定位 | 优先级 |
| ---- | ---- | ------ |
| Web 管理台 | 人类日常运营节点 | P0 |
| OpenAPI/API 文档 | AI、脚本、第三方集成的稳定入口 | P0 |
| 薄 CLI | 安装、服务生命周期、少量脚本化操作 | P1 |
| MCP/Agent 工具 | 让 AI 原生管理节点、订单和审计 | P2 |

CLI 不作为主要产品形态。AI 友好性的核心是结构化 API、稳定错误码、幂等操作和可查询状态，而不是把所有功能都复制进 CLI。

## 三、现有代码基础

| 模块       | 状态    | 关键实现                                                      |
| ---------- | ------- | ------------------------------------------------------------- |
| P2P 组网   | 已完成  | Hyperswarm 4.x 双 swarm（文件+聊天），DHT bootstrap 兼容 Keet |
| 内容存储   | 已完成  | Hyperdrive 13.x + CID v1 UnixFS，内容寻址                     |
| 文件管理   | 已完成  | 上传/下载/预览/回收站/收藏/移动重命名                         |
| 聊天频道   | 已完成  | Hypercore 日志 + P2P 同步 + WebSocket 推送                    |
| 身份系统   | 已完成  | PBKDF2 派生钱包，EVM 地址 + Ed25519/X25519 密钥               |
| 桌面客户端 | 已完成  | Electron 41，Win/Mac/Linux 打包                               |
| 浏览器 demo | 已完成  | Next.js 16 静态导出，响应式 Liquid Glass 设计                 |

v1 长期存储节点以服务器 daemon 为主，日常运营通过 Web 管理台完成；桌面端复用同一套 API 作为个人控制台。浏览器公共站点只展示 demo、文档和客户端下载引导，不作为长期存储节点。

### 核心架构文件

```
server/src/index.js             # MostBoxEngine 核心类
server/src/core/cid.js          # UnixFS CID v1 计算与链接解析

server/index.js                 # Hono HTTP 服务 + WebSocket + API 路由
```

## 五、非 MVP 能力

| 能力 | 当前决策 |
| ---- | -------- |
| 默认应用层加密 | 暂不内置；用户可自行本地加密后上传 |
| 文件分片/纠删码 | 暂不实现；MVP 采用用户自选数量的完整副本，replicaCount >= 1 |
| 浏览器正式节点能力 | 暂不承诺；浏览器只展示 demo 并引导下载客户端 |
| 完整 CLI 产品 | 暂不实现；只做安装、启动、诊断用薄 CLI |
| 原生代币 / DAO 治理 | 暂不实现；v1 使用 USDT 和多签/参数管理员 |
| 多稳定币 | 暂不实现；v1 仅支持 USDT |
| 自动抽签 | 暂不实现；用户手动选择节点 |
| 自动 repair | 暂不实现；先标记 Degraded，再手动替换节点 |
| ERC-4337 Paymaster | 暂不实现；MVP 不做 gas 代付 |
| 同 CID 多活跃订单 | 暂不实现；v1 同一 CID 同时只有 1 个 active order |
| 复杂金库分账 | 暂不实现；手续费和罚没余额先进单一 Treasury 地址 |
| x402 | 暂不考虑；MVP 仍以链上订单和 USDT 托管为主 |
| 零知识证明 | 暂不实现；MVP 使用 Merkle 审计 + fraud proof |
