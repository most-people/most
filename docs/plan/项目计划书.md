# MostBox 项目计划书

> P2P 文件分享、内容寻址、默认做种、凭 `most://` 链接下载并校验。

---

## 项目摘要

| 项目 | 内容 |
| ---- | ---- |
| 名称 | MostBox |
| 核心定位 | 类 BitTorrent / 磁力链接的 P2P 分享工具 |
| 分享入口 | `most://<cid>?filename=...&r=<chunkMerkleRoot>` |
| 技术底座 | Hyperswarm + Hyperdrive + UnixFS CID v1 + Merkle Tree |
| 保留能力 | Web3 工具箱作为独立入口，不参与文件分享主流程 |
| 明确不做 | 上链存储协议、支付、质押、订单、赏金猎人、fraud proof、Treasury、云端下单 |

MostBox 的核心诉求不是替用户备份数据，而是在用户在线的前提下把文件传播出去。每个完成下载的人默认继续做种，只要仍有 peer 在线，后来的人就能继续下载。

## 一、市场痛点

| 场景 | 痛点 |
| ---- | ---- |
| 网盘分享 | 依赖中心平台，可能限速、封禁、过期、需要账号 |
| 传统磁力/BT | 体验老旧，缺少现代桌面端、文件管理和本地校验展示 |
| IPFS 使用 | 对普通用户不够直观，常需要网关或额外 pinning 概念 |

MostBox 的切入点是：把“发链接、下载、自动做种、校验文件”做成普通人能理解的客户端体验。

## 二、核心价值主张

| 价值 | 实现方式 |
| ---- | -------- |
| 内容寻址 | 相同内容得到相同 CID，链接携带完整性锚 |
| 无账号分享 | 知道 `most://` 链接即可下载；CID 即权限 |
| 默认传播 | 发布和下载成功后默认做种 |
| 本地校验 | 下载后重算 CID 与 `chunkMerkleRoot` |
| 完整副本 | 不做纠删码、不做分片；每个种子持有完整文件 |
| 用户自控 | 用户自己保留重要数据，可暂停或删除本机做种 |

MostBox 不是隐私网盘。默认文件按公共内容处理；私密文件必须由用户先本地加密，再把密文作为普通文件分享。

## 三、核心流程

### 3.1 发布与分享

```
用户选择文件
  → 计算 UnixFS CID v1
  → 按 256KB chunk 计算 chunkMerkleRoot
  → 写入 Hyperdrive，key = /<cid>
  → join topic = cid.multihash.digest
  → 生成 most:// 链接
  → 本机保持做种
```

### 3.2 下载与继续做种

```
下载者输入 most:// 链接
  → 解析 cid、filename、r
  → join topic = cid.multihash.digest
  → 从可用 peer 拉取完整文件
  → 重算 CID 与 chunkMerkleRoot
  → 校验通过后写入本机 Hyperdrive
  → 默认加入做种列表
```

### 3.3 热门文件传播

热门文件越多人下载，在线种子越多：

```
发布者 A → 下载者 B
发布者 A + B → 下载者 C
B + C → 下载者 D
...
```

发布者退出后，网络不保证永久可用；但只要 B/C/D 任意一个仍在线做种，新用户仍可下载。

## 四、产品边界

| 类型 | 决策 |
| ---- | ---- |
| 云端上传 | 不做；MostBox 不提供中心化上传或托管 |
| 付费保种 | 不做；没有订单、支付、结算或赔付 |
| 长期备份 | 不承诺；用户自己保存重要数据 |
| 默认加密 | 不做；用户可自行加密后分享密文 |
| Web3 | 保留独立工具箱，不进入分享主流程 |
| 浏览器 | 只展示 demo、文档和下载入口；正式分享/做种在桌面端或 daemon |

## 五、竞品对比

| 项目 | 分享方式 | 可用性来源 | 校验 | 中心依赖 |
| ---- | -------- | ---------- | ---- | -------- |
| 网盘 | 平台链接 | 平台存储 | 平台负责 | 强 |
| BitTorrent | magnet/torrent | 做种 peer | piece hash | 弱 |
| IPFS | CID | provider/pinner | CID | 弱 |
| MostBox | `most://` | 在线做种 peer | CID + Merkle root | 弱 |

## 六、风险与应对

| 风险 | 应对 |
| ---- | ---- |
| 没有种子就无法下载 | UI 展示“需要至少一个 peer 在线”，不承诺备份 |
| 链接泄露导致下载 | 明确 CID 即权限；私密文件先本地加密 |
| 恶意 peer 返回坏数据 | 下载后必须 CID + `chunkMerkleRoot` 双重校验 |
| 热门文件占满上行 | seed policy 支持最大同时做种数和上传限速 |
| 用户误以为是云盘 | 文案统一为“分享/做种”，删除“云端订单/赔付/保种市场”叙事 |

## 七、总结

MostBox 的 v1 目标是把现有 P2P 底座收敛成一个可靠的分享工具：一个文件、一个 CID、一棵 Merkle 树、一个 Hyperswarm topic、一个 `most://` 链接。用户在线时发布，下载者完成后默认接力做种，文件靠在线 peer 传播，而不是靠支付市场或链上承诺保证。
