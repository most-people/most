# 独立工具箱与笔记计划

> MOST PEOPLE 可以是去中心化工具箱，但 MostBox 文件分享 MVP 仍只按 `most://`、CID 校验和默认做种闭环验收。

## 一、定位

当前仓库包含多个入口：

| 入口     | 定位                                   | 是否进入 MostBox 文件分享主线 |
| -------- | -------------------------------------- | ----------------------------- |
| `/app`   | MostBox P2P 文件分享、下载和默认做种   | 是                            |
| `/admin` | 做种节点管理台                         | 是                            |
| `/note`  | 本地 Markdown 笔记、私密内容和加密备份 | 否                            |
| `/web3`  | 账户、钱包和密钥工具                   | 否                            |
| `/chat`  | P2P 频道聊天                           | 否                            |
| `/ping`  | 网络和工具展示页                       | 否                            |

独立工具可以共享登录、主题、UI shell、Web3 密钥或 P2P 基础设施，但不能成为发布文件、下载文件或继续做种的前置条件。

## 二、笔记当前状态

| 能力          | 当前实现                                                | 入口                                                                          |
| ------------- | ------------------------------------------------------- | ----------------------------------------------------------------------------- |
| Markdown 编辑 | Milkdown / Crepe 块编辑器，支持阅读和编辑模式           | `components/MilkdownEditor.tsx`                                               |
| 本地保存      | Zustand + `localStorage`，key 为 `mostbox_notes`        | `app/app/useAppStore.ts`                                                      |
| 路径和文件夹  | 通过笔记 `path` 推导目录，支持重命名、移动和删除目录    | `server/src/utils/noteUtils.js`                                               |
| 搜索          | 按笔记名称搜索                                          | `app/note/page.tsx`                                                           |
| 私密笔记      | 使用 `mostEncode` / `mostDecode` 和 Web3 登录态加密内容 | `app/note/page.tsx`                                                           |
| 云备份        | 上传加密后的 notes payload，使用钱包签名认证            | `server/src/utils/noteBackup.js`                                              |
| 本地导入导出  | 导出加密文本，导入时覆盖本地笔记并可触发同步            | `app/note/useNoteBackupSync.ts`                                               |
| 单测          | 覆盖路径、名称、raw CID、备份加密和签名 header          | `server/tests/unit/noteUtils.test.js`、`server/tests/unit/noteBackup.test.js` |

## 三、边界

| 问题                                      | 决策                                       |
| ----------------------------------------- | ------------------------------------------ |
| 笔记是否属于 MostBox 文件分享 MVP         | 不属于；它是独立工具                       |
| 笔记云备份是否代表 MostBox 提供云盘       | 不代表；只覆盖笔记应用数据                 |
| 私密笔记是否改变 MostBox 默认公开分享模型 | 不改变；文件分享仍是 CID 即权限            |
| `calculateNoteCid()` 是否可用于文件分享   | 不可；它是 raw CID，只用于笔记和备份       |
| 笔记是否需要后端 daemon                   | 基础本地编辑不需要；云备份调用外部备份 API |

## 四、近期任务

| 优先级 | 任务             | 验收                                             |
| ------ | ---------------- | ------------------------------------------------ |
| P1     | 补齐笔记边界文案 | 用户不会把笔记云备份理解成 MostBox 文件云盘      |
| P1     | 私密笔记回归     | 未登录不可解密，正确登录后可阅读、编辑、重新保存 |
| P1     | 备份状态回归     | 云端缺失、冲突、失败、本地导入导出都有可读反馈   |
| P1     | 资源管理回归     | 新建、重命名、移动、删除文件夹、搜索不丢数据     |
| P2     | 备份 API 观察    | 记录失败率、错误文案和恢复路径                   |

## 五、验证命令

```bash
node --test server/tests/unit/noteUtils.test.js server/tests/unit/noteBackup.test.js
```

当前 `npm run test:unit` 会跑全部单测；只改笔记工具时，优先跑上面两个文件或直接跑 `npm run test:unit`。
