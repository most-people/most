# MostBox 节点运营规格

> 以 `项目计划书.md` 为准：存储节点贡献硬盘与带宽，质押 MOST，监听链上订单，报价报名，入选后持有完整文件副本并赚取稳定币收益。

---

## 一、节点角色

存储节点是 MostBox 网络的资源提供者，负责保存完整文件副本并向下载者提供 P2P 上传带宽。

### 核心职责

- 质押 MOST 作为保证金，声明可用容量。
- 监听 `OrderCreated` 事件，提交报价、容量和信誉承诺。
- 入选后 join 订单 topic，pull 完整文件，本地验证 chunkMerkleRoot 后提交 `pullConfirm`。
- 订单有效期内持续提供下载。
- 可充当赏金猎人审计其他节点。

### 收益来源

- 存储订单费用：用户支付稳定币，90% 分给订单存储节点。
- 赏金审计：发现欺诈后获得被罚没质押的 30%。
- 早期节点激励：来自 MOST 节点质押激励池。

## 二、节点运营流程

### 2.1 注册与报价

```
部署节点客户端 → 配置存储目录和容量上限 → 质押 MOST → NodeRegistry 注册
  → 接入 Hyperswarm 网络 → 监听 OrderCreated → 对合适订单提交报价
```

### 2.2 订单接收与存储

```
OrderCreated
  │
  ├── 节点读取 {cid, topic, chunkMerkleRoot, chunkSize, m}
  ├── 提交报价、容量和信誉承诺
  ├── 等待用户选择或自动抽签
  ├── 入选后 join(topic) → pull 完整文件 → 本地重算 chunkMerkleRoot
  ├── submitPullConfirm(orderId, chunkMerkleRoot)
  └── 成为订单存储节点
```

合约不按响应速度直接分配节点。节点靠价格、容量、信誉、带宽和质押资格进入候选池，最终由用户选择或自动策略抽签。

### 2.3 收益结算

- 用户每个订单预付 1 个月稳定币费用。
- 到期后若无 fraud proof，90% 费用释放给 m 个存储节点平分。
- 10% 进入网络手续费，其中 70% 回购销毁 MOST，30% 进入 Repair 储备金。
- 节点丢失数据并被 fraud proof 证明后，按最低罚没额/比例/重复违规系数罚没。
- 单次审计不响应扣当期可结算收益/信誉分；多次独立审计不响应则冻结收益并触发 repair。

---

## 三、信誉分

信誉分不负责订单匹配，只用于展示、治理参数和风控参考。

```
信誉分 = 基础分 100
  - fraud proof 成立: -50 分
  - 单次不响应审计: -5 分/次
  - 多次独立审计不响应: 冻结收益并触发 repair
  - repair 失败: -20 分
  + 连续在线 30 天: +5 分
  + 订单到期正常结算: +1 分/单
```

## 四、节点部署

### Docker 一键部署

```bash
docker run -d \
  --name mostbox-node \
  -v /data/mostbox:/data \
  -p 1976:1976 \
  -e MOSTBOX_MODE=node \
  -e MOSTBOX_STAKE=100000 \
  mostbox/node:latest
```

### 系统要求

| 资源 | 最低要求        | 推荐配置    |
| ---- | --------------- | ----------- |
| CPU  | 2 核            | 4 核        |
| 内存 | 4 GB            | 8 GB        |
| 存储 | 100 GB SSD      | 1 TB NVMe   |
| 带宽 | 10 Mbps         | 100 Mbps    |
| 网络 | 公网 IP 或 UPnP | 固定公网 IP |

## 五、P2P 安全

| 风险 | 应对 |
| ---- | ---- |
| 恶意节点 | MOST 质押 + fraud proof 最低罚没额/比例/重复违规系数 |
| Sybil 攻击 | 质押门槛 + 容量上限 + 信誉分 |
| 数据篡改 | CID 完整性校验 + chunkMerkleRoot |
| 谎报容量 | 存不下订单会在审计/下载中暴露并触发罚没 |
| 热门文件带宽压力 | 下载者完成后临时做种，分摊上传压力 |
