# MostBox 节点运营规格

> 以 `项目计划书.md` 为准：存储节点贡献硬盘与带宽，质押 MOST，监听链上订单，先到先得抢单，持有完整文件副本并赚取稳定币收益。

---

## 一、节点角色概述

存储节点是 MostBox 网络的资源提供者，负责保存完整文件副本并向下载者提供 P2P 上传带宽。

### 核心职责

- 质押 MOST 作为保证金。
- 声明可用容量，容量决定可接订单上限。
- 监听 `OrderCreated` 事件。
- join 订单 topic，从上传者或已有 peer pull 完整文件。
- 本地验证 Merkle root 后提交 `pullConfirm`。
- 订单有效期内持续提供下载。
- 遇到节点失败时参与 repair，全量拷贝完整副本。

### 收益来源

- 存储订单费用：用户支付稳定币，90% 分给订单存储节点。
- 赏金审计：节点也可充当赏金猎人，发现欺诈后获得被罚没质押的 30%。
- 早期节点激励：来自 MOST 节点质押激励池。

## 五、节点运营流程

### 5.1 节点注册

```
部署节点客户端
  → 配置存储目录和容量上限
  → 质押 MOST
  → NodeRegistry 注册
  → 接入 Hyperswarm 网络
  → 监听 OrderCreated
```

### 5.2 订单接收与存储

```
OrderCreated
  │
  ├── 节点读取 {CID, digest, merkleRoot, chunkSize, m}
  ├── join(digest)
  ├── pull 完整文件
  ├── 本地重算 Merkle root
  ├── submitPullConfirm(orderId, merkleRoot)
  └── 若属于前 m 个确认者，则成为存储节点
```

合约不预先匹配节点。节点靠监听事件、网络速度、容量和质押资格参与先到先得抢单。

### 5.3 收益结算

- 用户每个订单预付 1 个月稳定币费用。
- 到期后 90% 费用释放给 m 个存储节点平分。
- 10% 进入网络手续费，其中 70% 回购销毁 MOST，30% 进入 Repair 储备金。
- 节点丢失数据并被 fraud proof 证明后，100% 质押罚没。

## 六、赏金审计

任何节点都可以审计其他订单节点：

```
审计者
  │
  ├── 查询订单节点列表
  ├── join(topic)
  ├── 请求目标节点返回 chunk[i] + Merkle proof
  ├── 本地验证 proof
  └── 失败时提交 fraud proof
```

| 结果 | 后续 |
| ---- | ---- |
| proof 有效 | 无链上操作 |
| proof 无效 | 上链提交 fraud proof，若成立则罚没被审节点 |
| 节点不响应 | 记录信誉报告，不直接罚没 |
| 诬告 | 审计者质押被罚，部分奖励给被诬节点 |

---

## 七、信誉分

信誉分不负责订单匹配，只用于展示、治理参数和风控参考。

```
信誉分 = 基础分 100
  - fraud proof 成立: -50 分
  - 多次不响应审计: -5 分/次
  - repair 失败: -20 分
  + 连续在线 30 天: +5 分
  + 订单到期正常结算: +1 分/单
```

## 八、节点部署

### 8.1 Docker 一键部署

```bash
docker run -d \
  --name mostbox-node \
  -v /data/mostbox:/data \
  -p 1976:1976 \
  -e MOSTBOX_MODE=node \
  -e MOSTBOX_STAKE=100000 \
  mostbox/node:latest
```

## 九、P2P 安全

| 风险 | 应对 |
| ---- | ---- |
| 恶意节点 | MOST 质押 + fraud proof 罚没 |
| Sybil 攻击 | 质押门槛 + 容量上限 + 信誉分 |
| 数据篡改 | CID 完整性校验 + Merkle root |
| 谎报容量 | 存不下订单会在审计/下载中暴露并罚没 |
| 热门文件带宽压力 | 下载者完成后临时做种，分摊上传压力 |
