# MostBox 节点运营规格

> 存储节点（矿工端）完整设计：模式切换、注册、订单、证明、收益

---

## 一、节点角色概述

存储节点是 MostBox 网络的核心参与者，贡献硬盘空间和带宽以赚取 MOST 代币收益。

### 核心职责

- 质押 MOST 作为保证金
- 自动接收并存储加密分片
- 定期提交零知识存储证明
- 保持高在线率以获取更高收益

### 收益来源

- 存储订单费用（90% 归节点）
- 节点等级奖励（高等级优先匹配高价值订单）
- 挖矿释放（60% 总量分配给节点）

## 五、节点运营流程

### 5.1 节点注册

```
部署节点客户端
  → 质押 MOST
  → 接入 Hyperswarm 网络
  → 合约分配订单
  → 接收并存储加密分片
  → 定期提交存储证明
  → 合约自动结算 MOST 收益
  → 随时提现
```

### 5.2 订单接收与存储

1. 节点监听 `OrderMatched` 事件
2. 合约根据节点资质（质押量、信誉、存储余量）匹配
3. 节点通过 P2P 接收加密分片
4. 存储完成后提交存储证明
5. 合约验证证明，订单生效

### 5.3 收益结算

- 收益按存储量、在线时长、信誉分计算
- 节点可随时调用 `claimRewards()` 领取
- 离线 >24h 扣当日收益
- 丢失数据扣质押金
- 多次违规永久封禁

## 六、节点等级系统

| 等级 | 条件                  | 权益                         |
| ---- | --------------------- | ---------------------------- |
| Lv1  | 新注册节点            | 基础匹配优先级               |
| Lv2  | 信誉 > 80，在线 30 天 | 5% 价格折扣，优先匹配        |
| Lv3  | 信誉 > 95，在线 90 天 | 10% 价格折扣，高价值订单优先 |
| Lv4  | 顶级节点              | 15% 价格折扣，最高优先级     |

### 信誉分计算

```
信誉分 = 基础分 100
  - 离线 >24h: -5 分/次
  - 证明失败: -10 分/次
  - 数据丢失: -20 分/次
  - 连续在线 30 天: +5 分
  - 存储量达标: +2 分/月
```

## 七、节点部署

### 7.1 Docker 一键部署

```bash
docker run -d \
  --name mostbox-node \
  -v /data/mostbox:/data \
  -p 1976:1976 \
  -e MOSTBOX_MODE=node \
  -e MOSTBOX_STAKE=10000 \
  mostbox/node:latest
```

### 7.2 系统要求

| 资源 | 最低要求        | 推荐配置    |
| ---- | --------------- | ----------- |
| CPU  | 2 核            | 4 核        |
| 内存 | 4 GB            | 8 GB        |
| 存储 | 100 GB SSD      | 1 TB NVMe   |
| 带宽 | 10 Mbps         | 100 Mbps    |
| 网络 | 公网 IP 或 UPnP | 固定公网 IP |

## 八、P2P 安全

| 风险       | 应对                            |
| ---------- | ------------------------------- |
| 恶意节点   | 质押惩罚 + 信誉系统             |
| Sybil 攻击 | 质押门槛 + 工作量证明           |
| 数据篡改   | CID 完整性校验 + 多副本交叉验证 |
